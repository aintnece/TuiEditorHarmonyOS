# Phase 8.2-Ex175 — tight list 内块级子节点的换行（CommonMark 5.2/5.3）

## 目标
tight list item 内**非段落块级子节点**（HtmlBlock / 嵌套 List / CodeBlock / BlockQuote / Heading / ThematicBreak）前缺少换行 → `<li><div>` 应为 `<li>\n<div>`。基线 exact **82.67% (539/652)**。目标：Ex175 转正（HTML blocks 推到 44/44 满分）+ 可能连带嵌套列表例，全 26 节 0 回归。

## 只改一个文件
**`entry/src/main/ets/parser/html/Renderer.ts`** 的 `renderListItemChildren`。不动别处。

## 现状（要改的代码，Renderer.ts）
```ts
private renderListItem(node: AstNode): string {
  const isTight: boolean = node.attrs.tight === true;
  // 任务列表 ...
  const inner: string = this.renderListItemChildren(node, isTight);
  if (isTight) {
    return '<li>' + inner + '</li>\n';
  }
  return '<li>\n' + inner + '</li>\n';
}

private renderListItemChildren(node: AstNode, tight: boolean): string {
  let result: string = '';
  for (let i: number = 0; i < node.children.length; i++) {
    const child: AstNode = node.children[i];
    if (tight && child.type === AstNodeType.Paragraph) {
      result += this.renderChildren(child);   // tight 段落 unwrap，无 <p>
    } else {
      result += this.renderNode(child);        // ❌ tight 块级子节点前没补 \n
    }
  }
  return result;
}
```

## 现状 bug
- Ex175 `- <div>\n- foo`：第一项 tight、子节点是 HtmlBlock → `renderListItem` 走 tight 分支 `<li>` + inner + `</li>`，inner = `renderNode(HtmlBlock)` = `<div>\n` → 输出 `<li><div>\n</li>`。
- 期望 `<li>\n<div>\n</li>`（`<li>` 后、块级内容前要有 `\n`）。

## CommonMark 渲染规则
tight list item 里：
- **段落**子节点：unwrap（无 `<p>`、无包围换行）——保持现状。
- **非段落块级**子节点（HtmlBlock/List/CodeBlock/BlockQuote/Heading/ThematicBreak…）：前面要有一个**条件换行**（若已输出内容不以 `\n` 结尾、或还没输出过内容即紧跟 `<li>`，则补一个 `\n`）。块级子节点自身尾随的 `\n` 负责 `</li>` 前的换行。
- 例：
  - `[HtmlBlock]` → `<li>` + `\n` + `<div>\n` + `</li>` = `<li>\n<div>\n</li>`（Ex175）。
  - `[Paragraph, HtmlBlock]` → `<li>` + `foo` + `\n` + `<bar>\n` + `</li>` = `<li>foo\n<bar>\n</li>`。
  - `[Paragraph(a), List(b)]`（`- a\n  - b` tight）→ `<li>a\n<ul>\n<li>b</li>\n</ul>\n</li>`（连带可能修嵌套列表）。
  - `[Paragraph]`（普通 `- foo`）→ `<li>foo</li>`（无变化，25 个现有 tight 例不动）。

## 目标实现（renderListItemChildren）
```ts
private renderListItemChildren(node: AstNode, tight: boolean): string {
  let result: string = '';
  for (let i: number = 0; i < node.children.length; i++) {
    const child: AstNode = node.children[i];
    if (tight && child.type === AstNodeType.Paragraph) {
      // Tight 段落：unwrap，无 <p>，无包围换行
      result += this.renderChildren(child);
    } else {
      // 块级子节点（或 loose 模式）：tight 时块级子节点前补条件换行
      if (tight && (result.length === 0 || result[result.length - 1] !== '\n')) {
        result += '\n';
      }
      result += this.renderNode(child);
    }
  }
  return result;
}
```
> 关键：条件换行**只在 `tight` 时加**。loose 模式（`tight===false`）完全不变——loose item 由 `renderListItem` 的 `<li>\n` + 各块自带尾随 `\n` 处理，保持现状。tight 段落分支不变。

## 逐例验证逻辑
- **Ex175** `- <div>\n- foo`：项1 tight、HtmlBlock 子节点：result='' → 补 `\n` → `\n` + `<div>\n` → `<li>\n<div>\n</li>`；项2 `foo` 是 tight 段落 → `<li>foo</li>`。→ `<ul>\n<li>\n<div>\n</li>\n<li>foo</li>\n</ul>`。✓
- 普通 tight 列表（`- a\n- b`）：纯段落子节点 → 不进 else 分支 → 无变化。
- 嵌套 tight 列表：段落后接子 List → 补 `\n` → 可能转正（连带）。
