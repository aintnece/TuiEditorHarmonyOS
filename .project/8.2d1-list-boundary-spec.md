# Phase 8.2d-1 — 列表边界与编号正确性（contained，低风险）

只修列表的**边界划分 + 编号**，**不动列表项内容模型**（多块/嵌套/tight-loose 留 8.2d-2）。

## 现状 bug（Blocks.ts）
- `isOrderedListMarker`(438)：只认 `数字.`，**不认 `数字)`**（Ex302 的 `3) baz` 被当普通段落丢掉）。
- `parseList`(177)：循环里只要 `isXxxListMarker` 为真就继续，**不管 marker 字符/分隔符是否变化** → `- foo`/`+ baz` 被并成一个 `<ul>`（应是两个，Ex301）；`1.`/`3)` 不分家。
- 无 `start`：有序列表起始非 1（如 `3)`）应渲染 `<ol start="3">`，现只 `<ol>`。
- marker 剥离粗暴：有序 `indexOf('.')`、无序写死 `substring(2)`。

## 要实现

### 1) marker 识别（Blocks.ts，ArkTS：具名 class，禁匿名对象字面量！）
新增两个解析函数 + 具名结果 class（**不要返回 `{...}` 匿名字面量**，8.2c-1 踩过会炸 DevEco）：
```ts
class BulletMarker { ch: string = ''; contentStart: number = 0; }   // ch ∈ - * +
class OrderedMarker { num: number = 0; delim: string = ''; contentStart: number = 0; } // delim ∈ . )
function parseBulletMarker(line: string): BulletMarker | null   // 行首 [-*+] + ≥1 空白；contentStart=内容起始下标
function parseOrderedMarker(line: string): OrderedMarker | null // 行首 1-9 位数字 + (. 或 )) + ≥1 空白；num=数字值, delim=分隔符
```
- `isBulletListMarker`/`isOrderedListMarker` 改为基于上面两个函数（`isOrderedListMarker` 现在也认 `)`）。保持它们返回 boolean（parseParagraph 第263行的段落中断判定继续用）。

### 2) parseList(Blocks.ts) 边界 + start
- 进入时解析**首项** marker，记下「列表身份」：无序记 `ch`，有序记 `delim`；有序的 `num` 作为 `list.attrs.start`。
- 循环每行：解析 marker；若**无序且 ch 变了**、或**有序且 delim 变了**、或**有序↔无序切换** → **break**（结束本列表；state 停在变化行，外层 parseBlocks 会对它再起一个新列表 → 自然分家）。
- 首行内容剥离用 marker 的 `contentStart`（`line.substring(contentStart)`），替换 `indexOf('.')`/`substring(2)`。
- **子块收集逻辑（213-237 行）本批保持不动**（内容模型留 8.2d-2）；只把首行 marker 剥离改对。
- `list.attrs.start` 只对有序、且首项 num≠1 时有意义；无序忽略。

### 3) AST + 渲染（Node.ts + Renderer.ts）
- `AstAttrs` 加字段 `start: number = 1;`（若没有）。parseList 设 `list.attrs.start = 首项num`（有序）。
- `Renderer.renderList`(177-181)：有序且 `node.attrs.start !== 1` 时输出 `<ol start="N">`（N=start），否则 `<ol>`。无序不变。
