# 9.1 — 主题统一（B 全量变量化） · CC 实施 spec

> 上游：`.project/9.0-theme-unification-plan.md`（方案）+ `active-task.md`（约定）。
> 策略：用户拍板 **B 全量变量化**——三处暗色（WYSIWYG / 预览 / 外壳）颜色全部由 ThemeService token 派生，硬编码 → CSS 变量。
> 验证：**真机**（commonmark harness 不覆盖 UI）。CC 不编译，push 后由用户 DevEco 真机验。

---

## 0. 背景结论（精确定位已完成）

- 割裂元凶 = **WYSIWYG**：`editor.html` 的 `__ww.setTheme(dark)` 只「启用 `toastui-editor-dark.css` + 切 body class」，**完全没喂 ThemeService token**。tui dark.css 473 行全硬编码、零 CSS 变量，用的是另一套蓝灰阶（bg `#232428`/深 `#121212`/accent `#67ccff`），与外壳 VSCode 调色板（`#1e1e1e`/`#d4d4d4`/`#569cd6`）对不上。
- 预览侧 `Renderer.ts → renderFullPage()` 是**我们自己的模板字符串**，暗色硬编码本来就≈外壳 token，只需顺手变量化、防漂移。
- 注入通道现成：WYSIWYG 走 `__ww.setTheme`；预览走 `MdPreview` 的 `stateChange` 整页重载 + `getFullHTML()`。

## 1. 工程判断（已定，CC 照做）

1. **prism 语法高亮色不动**：`prism-tomorrow.min.css` / `prism-light.min.css` 保持原样。关键字/字符串/注释是语义着色，不统一到外壳灰度。只统一代码块**背景**（由 tui 的 `pre` / `*-code-block-line-background` 控制，已纳入变量化）。
2. **行内代码 `code` 中性化**：tui 暗色的粉红字 `#c1798b` / 深红底 `#35262a` 归一到中性（`--ed-fg` / `--ed-bg-elevated`），与外壳一致。
3. **base64 SVG 图标不动**：dark.css 里 task-list / image-link / code-block 按钮的 `url('data:image/svg+xml;base64,...')` 颜色烧在 base64 内，本批不拆（次要、复选框等）。保留原样。
4. **看不见的选择器低优先级**：本项目 editor.html 配置了 `hideModeSwitch:true` + `toolbarItems:[]`（内置工具栏 `display:none`）+ 纯 `wysiwyg`，所以 dark.css 中 toolbar / mode-switch / md-tab / md-syntax / popup / tooltip / switch / scroll-sync / custom-block 等选择器**在本应用渲染不到**。仍机械替换成就近变量（保持「全量变量化」彻底性），但**真机只重点验「内容区」**（见 §7）。

## 3. ThemeService.ts 改造（`services/ThemeService.ts`）

### 3.1 新增 3 个字段（`ThemeColors` class，editor 组附近）
```ts
editorCodeBg: string = '#f6f8fa';        // 行内+块代码背景（中性）
editorBorderSubtle: string = '#eaecef';  // 次级边框（引用左边框等）
editorAccentHover: string = '#0256b9';   // 链接/按钮 hover
```
- 默认值即浅色（LightTheme 沿用默认）。
- **DarkTheme** 补：`editorCodeBg='#2d2d2d'; editorBorderSubtle='#333333'; editorAccentHover='#4b8bc4';`
- **`copyFrom`** 逐字段同步这 3 个新字段（保持 ArkTS 显式赋值风格，**禁** spread/Object.assign）。

### 3.2 新增序列化方法 `toCssVars()`（`ThemeColors` class 内）
生成统一变量字符串，供两个 WebView 注入。**纯字符串拼接，禁对象/Record/索引访问**：
```ts
/** 序列化为 :root CSS 变量声明体（不含 :root{} 包裹），供 WebView 注入 */
toCssVars(): string {
  let s: string = '';
  s += '--ed-bg:' + this.editorBg + ';';
  s += '--ed-bg-elevated:' + this.editorCodeBg + ';';
  s += '--ed-bg-deep:' + this.editorGutter + ';';
  s += '--ed-fg:' + this.editorFg + ';';
  s += '--ed-fg-muted:' + this.editorLineNumber + ';';
  s += '--ed-fg-subtle:' + this.editorPlaceholder + ';';
  s += '--ed-border:' + this.borderColor + ';';
  s += '--ed-border-subtle:' + this.editorBorderSubtle + ';';
  s += '--ed-accent:' + this.editorCursor + ';';
  s += '--ed-accent-hover:' + this.editorAccentHover + ';';
  s += '--ed-selection:' + this.editorSelection + ';';
  s += '--ed-error:' + this.errorColor + ';';
  return s;
}
```

## 5. editor.html 改造（`resources/rawfile/tui-editor/editor.html`）

### 5.1 `<head>` 加一个空的变量样式块（供 JS 写入）
在现有 `<style>` 之前/之后加：
```html
<style id="ed-theme-vars"></style>
```

### 5.2 扩展 `__ww.setTheme` 签名 → `setTheme(dark, varsCss)`
```js
setTheme: function (dark, varsCss) {
  var darkLink = document.getElementById('tui-dark-css');
  if (darkLink) { darkLink.disabled = !dark; }
  var prismDark = document.getElementById('prism-dark-css');
  if (prismDark) { prismDark.disabled = !dark; }
  if (dark) { document.body.classList.add('toastui-editor-dark'); }
  else { document.body.classList.remove('toastui-editor-dark'); }
  // 注入/更新统一主题变量
  if (typeof varsCss === 'string' && varsCss.length > 0) {
    var el = document.getElementById('ed-theme-vars');
    if (el) { el.textContent = ':root{' + varsCss + '}'; }
  }
}
```
- 末尾初始调用改为：从 URL query 读初始 vars 不现实 → 初始 vars 由 ArkTS 在 `applyInitial()` 注入（见 §6）。保留 `window.__ww.setTheme(queryTheme === 'dark')`（无 vars，仅切 link/class），随后 ArkTS 再带 vars 调一次。
- **注意**：light 模式也要注入变量（浅色侧 contents 默认 CSS 多数不带颜色，但变量化后的 dark.css 在 light 时 disabled，影响小；变量始终注入保证一致）。

## 6. WwEditor.ets 改造（`editor/markdown/WwEditor.ets`）

把 ThemeService 变量随 setTheme 一起传给 WebView。

### 6.1 主题切换 handler（`themeHandler`，约 L66-72）
```ts
this.themeHandler = (): void => {
  if (!editorContext.core || !self.isReady) return;
  const theme: ThemeColors = editorContext.core.getTheme();
  const dark: boolean = theme.isDark;
  if (dark === self.lastIsDark) return;
  self.lastIsDark = dark;
  const vars: string = theme.toCssVars();
  self.controller.runJavaScript(
    'window.__ww && window.__ww.setTheme(' + (dark ? 'true' : 'false') +
    ', decodeURIComponent("' + encodeURIComponent(vars) + '"))'
  );
};
```

### 6.2 初始注入（`applyInitial()`，约 L189-193）
把现有 setTheme 调用替换为带 vars 版本：
```ts
const theme: ThemeColors = this.themeColors;
const vars: string = theme.toCssVars();
this.controller.runJavaScript(
  'window.__ww && window.__ww.setTheme(' + (theme.isDark ? 'true' : 'false') +
  ', decodeURIComponent("' + encodeURIComponent(vars) + '"))'
);
```
> `@Prop themeColors: ThemeColors` 已是完整对象，含新增字段；`toCssVars()` 是其方法，可直接调。

## 7. Renderer.ts 改造（`parser/html/Renderer.ts` → `renderFullPage`）— 预览侧变量化

把 §2 的 `:root` 变量注入预览页 `<head>`，body CSS 改读变量。**预览只有 dark/light 两态，变量值由 darkMode 决定**（Renderer 拿不到 ThemeService 实例，按 darkMode 内联两套值即可——值与 §2 表一致）。

### 7.1 在 `<style>` 顶部注入 `:root`
```ts
const root: string = this.options.darkMode
  ? ':root{--ed-bg:#1e1e1e;--ed-bg-elevated:#2d2d2d;--ed-bg-deep:#252526;--ed-fg:#d4d4d4;--ed-fg-muted:#858585;--ed-fg-subtle:#6a737d;--ed-border:#404040;--ed-border-subtle:#333333;--ed-accent:#569cd6;--ed-accent-hover:#4b8bc4;--ed-selection:#264f78;--ed-error:#f85149;}'
  : ':root{--ed-bg:#ffffff;--ed-bg-elevated:#f6f8fa;--ed-bg-deep:#f6f8fa;--ed-fg:#24292e;--ed-fg-muted:#6a737d;--ed-fg-subtle:#959da5;--ed-border:#dfe2e5;--ed-border-subtle:#eaecef;--ed-accent:#0366d6;--ed-accent-hover:#0256b9;--ed-selection:#c8e1ff;--ed-error:#d73a49;}';
```
（拼进模板：`<style>${root} * { box-sizing... }`）

### 7.2 body 段硬编码三元 → var()
- `body { background: var(--ed-bg); color: var(--ed-fg); ... }`
- `h1/h2 border-bottom: 1px solid var(--ed-border);`
- `code/pre/th { background: var(--ed-bg-elevated); }`
- `blockquote { border-left: 4px solid var(--ed-accent); background: var(--ed-bg-deep); }`
- `th,td { border: 1px solid var(--ed-border); }`
- `a { color: var(--ed-accent); }` `a:hover { color: var(--ed-accent-hover); }`
- `hr { border-top: 1px solid var(--ed-border); }`
- 删掉 `bg`/`fg`/`codeBg` 三个局部 const（不再用）。

> 这样预览与 WYSIWYG 用**同名变量同值**，外加外壳同源 → 三处灰度统一。

## 9. ArkTS 严格模式（审 diff 必过，CLAUDE.md 规则）

- `toCssVars()` **纯字符串 += 拼接**，禁对象字面量/Record/索引访问/spread。
- ThemeService 新字段在 `ThemeColors` 具名 class，`copyFrom` 逐字段显式赋值。
- WwEditor.ets 用具名 `const theme: ThemeColors`，调 `theme.toCssVars()`；禁匿名对象。
- Renderer 模板字符串内不引入 `\\`、不用正则。
- editor.html 是普通 JS（rawfile，不过 ArkTS 编译），但保持 ES5 风格（var/function）与现有一致。

## 10. 真机验证清单（用户在 DevEco 跑）

**核心（必验）**——切到深色，对比三处灰度是否一致：
1. WYSIWYG 编辑区背景 == 外壳背景（不再是发蓝的 `#232428`）。
2. WYSIWYG 正文/标题文字、表格、列表、引用左边框、分割线 灰度与预览/外壳协调。
3. WYSIWYG 链接色 == 外壳 accent（`#569cd6` 系），hover 变深。
4. 行内代码不再是粉红，变中性灰底。
5. 代码块**背景**统一，但语法高亮（prism）彩色保留、可读。
6. Markdown 预览深色与 WYSIWYG、外壳三者一致。
7. 浅色模式同样三处一致、无回归。
8. 深↔浅切换实时生效（WYSIWYG 走 setTheme、预览走整页重载）。

**次要（扫一眼）**：表格选中/单元格高亮、任务列表复选框（SVG 图标本批不改，颜色可能略不同属已知）。
