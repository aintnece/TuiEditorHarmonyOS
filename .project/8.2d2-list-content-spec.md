# Phase 8.2d-2 — 列表项内容模型重写（多块/嵌套/tight-loose）

接 8.2d-1（边界+编号已对）。本批重写**列表项内容收集**，根治 Ex253/254 内容错位 + 支持多块项/嵌套列表 + tight/loose。**风险较高**，严格守回归。

## 现状根因（Blocks.ts parseList 约 188-247）
1. **顺序 bug**：先把 subContent `parseBlocks` 进 item，**最后**才 `parseInlines(itemText)` 追加 → 首行被甩到末尾（Ex254）。
2. **首行只当单行 itemText**：段落续行被丢进 subContent 当独立块。
3. **去缩进用 `trimStart()` 全剥** → 毁掉嵌套相对缩进（嵌套列表/缩进代码块坏）。
4. 无 tight/loose → `<p>` 包裹恒错。

## 正确算法（按 CommonMark）

### A) 内容缩进列 W（扩展 8.2d-1 的 marker 解析）
marker 之后数空格数 `s`（tab 按制表位4 展开成列）：
- `s === 0`（marker 后即行尾/空行）→ W = markerWidth + 1
- `1 ≤ s ≤ 4` → W = markerWidth + s
- `s ≥ 5` → W = markerWidth + 1（其余空格属内容缩进，使内容成缩进代码块）
让 `parseBulletMarker`/`parseOrderedMarker` 的 `contentStart` 返回真实 W（现在写死 +1，要改成按上面数空格）。

### B) 收集整项内容（替换 188-247 的收集逻辑）
对每个列表项：
1. 首行内容 = `line.substring(W)`（**不要 trimStart**，保留相对缩进）。`state.nextLine()`。
2. 循环收集后续行进 `itemLines`（字符串数组，**保持顺序**）：
   - **空行** → 记一个空行（push ''），并标记「本项见过空行」（供 loose 判定）；继续。
   - 行按列展开后**缩进 ≥ W** → 去掉前 W 列（用 8.2c-1 风格的列感知去缩进；tab 展开）后 push；继续。
   - 行**缩进 < W 且非空**：
     - 若是**懒续行**（lazy continuation：该行不是新块起始——非 marker、非 `#`/```/`>`/thematic/html-block——且上一条收集行是段落文本）→ push 原行（不去缩进）；继续。
     - 否则 → **本项结束**（break，不消费该行）。
   - 遇到**新列表 marker 且缩进 < W** → 本项结束（break，不消费）。
3. 去掉 itemLines **尾部空行**（尾部空行不计入项内容，但若尾部空行后还有属于本列表的项，则该空行使列表 loose）。
4. 任务列表检测（GFM）：对**首行内容**判 `[ ] `/`[x] `/`[X] `（现有逻辑迁移到这里，作用于 itemLines[0]）。
5. 把 `itemLines.join('\n')` 作为该项内容，`new ParseState().reset(content)` + `parseBlocks(innerState, item)` —— **唯一**的内容解析路径（删掉原先单独的 `parseInlines(itemText)` 追加，顺序 bug 随之消失）。
6. 若 item 无子节点（空项）→ 留空即可（renderListItem 输出 `<li></li>`）。

### C) tight / loose 判定（parseList 层）
- **loose** 条件：① 任意两项之间有空行；② 任意一项内部、两个块级元素之间有空行（即项内容里出现「非尾部空行」）。否则 **tight**。
- 实现：收集每项时记录「项内是否含非尾部空行」+「项之间是否隔空行」；任一为真 → 整个列表 loose。
- 设 `list.attrs.tight = !loose`，并**把 tight 同样写到每个 ListItem 的 attrs**（`item.attrs.tight = list.attrs.tight`），供渲染器读取（renderListItem 拿不到父 list 上下文）。
- `AstAttrs` 若无 `tight` 字段则加 `tight: boolean = true;`（Node.ts）。

### D) 渲染器 tight 解包（Renderer.ts renderListItem 约 183-193）
- **loose**（`node.attrs.tight === false`）：照旧——子节点正常渲染，段落带 `<p>`。
- **tight**（`node.attrs.tight === true`）：对 item 的**直接子节点**，若是 `Paragraph` 节点 → 只渲染其 inner（`renderChildren(para)`，**不包 `<p>`**）；其它块（CodeBlock/BlockQuote/嵌套 List 等）正常渲染。
  - 即 tight 项的「段落」不包 `<p>`，但嵌套列表/代码块仍正常。
- 任务列表分支（checked）保持，tight 同理不包 `<p>`。
