# Phase 8.2c-1 — 内联链接目标(destination)/标题(title)解析重写（快赢）

Hermes 已用 8.1 harness 采样 Links 失败聚类。本批次**只修内联链接 `[text](dest "title")` 的 destination/title 解析**，**不做引用式链接 `[text][ref]` / `[ref]:` 定义**（那是更大的 8.2c-2，需 ref 表）。

## 范围
**只改** `entry/src/main/ets/parser/commonmark/Inlines.ts` 里的 `tryParseLinkOrImage`(约 224-255 行) 及其 URL/title 解析 helper（`findFirstSpace` 等可重写/替换）。**不要动块解析、不要碰引用式链接、不要改其它文件。**

## 现状（bug）
现 `tryParseLinkOrImage`：取 `](` 后到匹配 `)` 之间整段为 urlPart，用 `findFirstSpace` 切第一个空格 → 前为 url 后为 title，再粗暴去引号。问题：
- ❌ 不认尖括号目标 `<dest>`：`[link](<>)` 给 `href="&lt;&gt;"`（应 `href=""`）。
- ❌ URL 里反斜杠转义不解析：`[link](\(foo\))` 给 `href="\(foo\)"`（应 `href="(foo)"`）。
- ❌ 切分太糙：`[link](/my uri)`（裸 URL 含空格、无合法 title）应**判定不是链接**（输出 `<p>[link](/my uri)</p>`），现错切成 url=`/my` title=`uri`。

## 要实现的 CommonMark 规则（destination + title）
`](` 之后按顺序：可选空白 → **destination** → （若有 title：必须 ≥1 个空白分隔）可选 title → 可选空白 → `)`。任何一步不合法 ⇒ 返回 null（不是链接，调用方会当普通文本）。

### destination 两种形式
1. **尖括号式 `<...>`**：从 `<` 读到第一个**未转义**的 `>`。
   - 中间**不能有换行**（有则非法 → null）。
   - 中间**不能有未转义的 `<`**（CommonMark：null）。
   - 内容即 URL，**允许空格**。处理反斜杠转义（`\X`→`X`，X 为 ASCII 标点）。
   - 例：`<>`→url=""；`</my uri>`→url=`/my uri`（空格后续会归一化成 `%20`，见下）。
2. **裸式**（不以 `<` 开头）：从当前位置读，遇到 **ASCII 空白** 或 **控制字符** 停止；允许**平衡的**圆括号（`(`/`)` 必须配对，未配对的 `)` 结束 destination）；处理反斜杠转义。
   - 例：`/url`→`/url`；`\(foo\)`→`(foo)`；`foo(and(bar))`→`foo(and(bar))`。
   - 若裸 destination 后**紧跟非空白非 `)` 字符**（无法解析为 title 起始）⇒ null。

### title（可选）
destination 后若还有内容：必须先有 **≥1 空白**，然后 title 是 `"..."` / `'...'` / `(...)` 之一（内部处理反斜杠转义；`(...)` 内不能有未转义 `(`）。title 后只允许空白 + `)`。
- 若 destination 后是非空白且非 `)` 且不构成合法 title ⇒ null。
- `[link](/my uri)`：裸 destination 读到空格停在 `/my`，但其后 `uri` 不是合法 title 形式（没有引号）⇒ **null**（不是链接）。

### URL 归一化（最小集）
- 反斜杠转义已在上面解析。
- 把 URL 里的**字面空格**百分号编码为 `%20`（满足 `</my uri>`→`/my%20uri`）。其它字符的完整百分号编码**本批不做**（复杂，留后续），只处理空格即可覆盖样例。
- 渲染端 `renderLink`(Renderer.ts) 现用 `escapeAttr`(=escapeHtml) 转义 `< > & "`——保持不动；URL 里的 `<>` 已在解析阶段不会出现（尖括号被剥离）。
