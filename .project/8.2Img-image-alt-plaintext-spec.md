# Phase 8.2-Img — 图片 alt 纯文本化（CommonMark 6.4 Images）

## 目标
图片 `alt` 属性当前存的是**原始括号 markdown 文本**，应改成括号内联内容的**纯文本**（强调/链接/嵌套图片等标记全部剥成文字）。基线 exact **80.83% (527/652)**，Images **11/22**。本批是 contained 系统性修复，目标：Images 上升、全 26 节 0 回归。

## 只改一个文件
**`entry/src/main/ets/parser/commonmark/Inlines.ts`**。不动 Renderer（它已读 `node.attrs.alt`，本批只改 alt 的值）、不动其它文件。

## 现状（要改的两处 + renderImage 参考）

`Inlines.ts` 行内图片创建（约 94-95 行）：
```ts
const node: AstNode = new AstNode(AstNodeType.Image);
node.attrs.alt = result.text;        // ❌ result.text 是原始括号 markdown（如 "foo ![bar](/url)"）
```
引用式图片创建（约 105-106 行）：
```ts
const imgNode: AstNode = new AstNode(AstNodeType.Image);
imgNode.attrs.alt = refImgResult.text;   // ❌ 同样是原始文本
```
`html/Renderer.ts` renderImage（**不改**，仅供参考）：
```ts
const alt: string = this.escapeAttr(node.attrs.alt || '');   // 直接用 attrs.alt
```

**bug**：`result.text` / `refImgResult.text` 是括号内的**原始 markdown**。CommonMark 要求 alt = 该内容**解析为内联后的纯文本**（emphasis/strong/strike/link 只取其文字、嵌套 image 取其 alt、code 取其代码文字、软/硬换行→`\n`）。

## CommonMark 规则
图片 alt 是括号内联内容的纯文本表示：
- `![foo *bar*](u)` → alt `foo bar`（强调标记剥掉）
- `![foo [bar](/url)](u2)` → alt `foo bar`（链接只留文字）
- `![foo ![bar](/url)](u2)` → alt `foo bar`（嵌套图片取其 alt）
- `![foo *bar*][]`（collapsed 引用，已能解析出 src/title）→ alt `foo bar`

## 目标实现（Inlines.ts）

### 新增两个 helper
```ts
/** 取节点子树的「纯文本」（用于图片 alt）：
 *  Text/Code/HtmlInline → 字面 text；Image → 其 attrs.alt（已是纯文本）；
 *  SoftBreak/HardBreak → '\n'；其余容器节点(Emph/Strong/Strike/Link…) → 递归子节点。 */
function plainTextOf(node: AstNode): string {
  let out: string = '';
  for (let k: number = 0; k < node.children.length; k++) {
    const c: AstNode = node.children[k];
    const t: AstNodeType = c.type;
    if (t === AstNodeType.Text || t === AstNodeType.Code || t === AstNodeType.HtmlInline) {
      out += c.text;
    } else if (t === AstNodeType.Image) {
      out += c.attrs.alt;
    } else if (t === AstNodeType.SoftBreak || t === AstNodeType.HardBreak) {
      out += '\n';
    } else {
      out += plainTextOf(c);   // Emph/Strong/Strike/Link 等容器递归
    }
  }
  return out;
}

/** 把括号原始文本解析为内联、再抽纯文本，作为图片 alt。 */
function extractAltText(rawText: string): string {
  const tmp: AstNode = new AstNode(AstNodeType.Paragraph);
  parseInlines(rawText, tmp);
  return plainTextOf(tmp);
}
```
> 说明：嵌套图片在 `parseInlines` 解析时，会经同样的图片创建路径、其 `attrs.alt` 已被 `extractAltText` 处理成纯文本，故 `plainTextOf` 对 Image 直接取 `attrs.alt` 即正确（无需再剥）。递归有限（按括号嵌套深度），无死循环风险。

### 改两处图片创建
```ts
// 行内图片（约 95 行）
node.attrs.alt = extractAltText(result.text);
// 引用式图片（约 106 行）
imgNode.attrs.alt = extractAltText(refImgResult.text);
```

### 确认 AstNodeType 成员名
实现前用 grep 核对 `Node.ts` 的枚举成员实际名（`Text`/`Code`/`HtmlInline`/`Image`/`SoftBreak`/`HardBreak`/`Emph`/`Strong`/`Strike`/`Link`），按真实名写（本项目 8.2e-6 已加 `HtmlInline`）。`AstNode.text`、`AstNode.attrs.alt`、`AstNode.children` 均已存在。

## ArkTS 硬约束
- 无正则；纯遍历 + 字符串拼接。
- 无匿名对象字面量（返回 `string`）。`for` 循环用数组下标 `node.children[k]` 合法（数组下标，非对象属性下标）。
- 无 Record / 无 `obj[key]` 对象属性下标（`c.type`/`c.text`/`c.attrs.alt` 是具名属性访问，合法）。
- import 在顶部（本批不新增 import；`parseInlines`/`AstNode`/`AstNodeType` 同文件/已导入）。
- ⚠️ `plainTextOf` 的 `switch`/`if` 用 `c.type === AstNodeType.X` 比较；别用对象索引。
