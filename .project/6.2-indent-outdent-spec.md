# Phase 6.2 — 缩进 / 减少缩进（indent / outdent）spec

## 目标

工具栏新增「缩进」「减少缩进」两个按钮，两种模式（Markdown / WYSIWYG）都生效。

- WYSIWYG：路由到 tui.editor 的 `indent` / `outdent` 命令（引擎已确认含这两命令）。
- Markdown 源码模式：对**当前行**增减行首缩进（缩进 = 行首加 2 个空格；减少缩进 = 删除行首至多 2 个空格，或 1 个 tab）。

## 涉及 4 处改动（三处代码 + 2 个新 SVG）

1. `/data/docs/obsidian-vault/鸿蒙开发/踩坑记录/UI渲染踩坑.md` — 按钮设计规范 + SVG 资产规范。
2. `/data/docs/obsidian-vault/鸿蒙开发/踩坑记录/ArkTS严格模式规则.md` — 严格模式（尤其：**不要用正则**，用 split/字符判断替代；回调里别裸用 this）。
3. 现有同类实现做模板：`components/Toolbar.ets` 的 `iconBtn` 用法、`editor/Editor.ts` 的 `mapToTui`、`editor/commands/commands/MarkdownCommands.ts` 的 `prefixLine`（行首定位 + EditorState 改写模板）。

### 改动 1：`components/Toolbar.ets` — 加两个按钮

用现有 `this.iconBtn(media, tip, action)`（普通点击按钮，**不是菜单**）。在「代码块」按钮（现 `exec('Codeblock')` 那行）之后、其后的 `Divider` 之前，插入两行：

```
this.iconBtn($r('app.media.tui_indent'), '缩进', () => { if (editorContext.editor) { editorContext.editor.exec('Indent'); } })
this.iconBtn($r('app.media.tui_outdent'), '减少缩进', () => { if (editorContext.editor) { editorContext.editor.exec('Outdent'); } })
```

（与列表/代码块同组，逻辑上属于"块级缩进"。）

### 改动 2：`editor/Editor.ts` — mapToTui 加两条路由

在 `mapToTui` 里、`HorizontalRule` 那条之后、`return null;` 之前插入：

```
if (name === 'Indent') { c.cmd = 'indent'; return c; }
if (name === 'Outdent') { c.cmd = 'outdent'; return c; }
```

（无 payload，照搬 Bold/Italic 那种单行写法。）

### 改动 3：`editor/commands/commands/MarkdownCommands.ts` — 新增两个命令并注册

照搬现有 `UndoableCommand` 结构（execute 返回新 EditorState，undo 返回 `state.snapshot()`）。行编辑模板参考 `prefixLine`（约 316 行）：用 `state.markdown` / `state.selectionStart`，从 selStart 往回走到行首（`md[i-1] === '\n'`），用 `state.snapshot()` 拿副本后改 `result.markdown` / `result.selectionStart` / `result.selectionEnd` / `result.isDirty = true`。

**IndentCommand**（name `'Indent'`）：
- 定位当前行首 lineStart。
- `result.markdown = md.substring(0, lineStart) + '  ' + md.substring(lineStart);`（行首插 2 个空格）
- 选区整体右移 2：`result.selectionStart = state.selectionStart + 2; result.selectionEnd = state.selectionEnd + 2;`
- `result.isDirty = true;`

**OutdentCommand**（name `'Outdent'`）：
- 定位当前行首 lineStart。
- 看 `md[lineStart]`：
  - 若是 `'\t'` → 删这 1 个字符（removed = 1）。
  - 否则数行首连续空格数 n（用 while 循环，**不要正则**），removed = min(2, n)；若 removed === 0 则原样返回 `state`（无操作，不脏）。
- `result.markdown = md.substring(0, lineStart) + md.substring(lineStart + removed);`
- 选区左移 removed，但**不得越过行首、不得为负**：`result.selectionStart = Math.max(lineStart, state.selectionStart - removed); result.selectionEnd = Math.max(lineStart, state.selectionEnd - removed);`
- `result.isDirty = true;`

**注册**：在 `getBuiltinCommands()` 的 `cmds.push(...)` 列表里追加 `cmds.push(IndentCommand); cmds.push(OutdentCommand);`。

### 改动 4：新建 2 个 SVG 图标

`resources/base/media/tui_indent.svg`、`resources/base/media/tui_outdent.svg`：
- Feather Icons 风格，24×24 viewBox，`stroke="currentColor"`，`fill="none"`，stroke-width 2，圆角线帽。
- 视觉：indent = 几条横排文本线 + 一个**指向右**的箭头（增加缩进）；outdent = 同样的文本线 + 一个**指向左**的箭头（减少缩进）。（参考 format-indent-increase / format-indent-decrease 的常见画法。）

## 真机验证（用户在 DevEco）

- Markdown 模式：某行打字、光标停该行 → 点「缩进」→ 行首多 2 空格；连点两次 → 4 空格；点「减少缩进」→ 逐次减 2 空格。
- WYSIWYG 模式：在**列表项**里点「缩进」→ 该项变成下一级嵌套；「减少缩进」→ 升回上一级。（注：tui 的 indent/outdent 主要作用于列表/引用项，普通段落可能无变化，属正常。）
- 两个新按钮图标显示正常、hover tooltip 正常。
