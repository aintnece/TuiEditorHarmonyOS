# Phase 8.2-G2 — 实体解码接入 dest / title / info（CommonMark 2.5 横切）

## 目标
G-1 已就位 HTML5 实体表 + `tryParseEntity`（命名表查找 + 数字）。本批把**实体 + 反斜杠转义的单趟解码 `unescapeString`** 接到：链接/图片 **destination**、**title**、代码围栏 **info string**。基线 exact **81.75% (533/652)**。目标：Ex32/33/34 转正 + 连带 Links/Images/Link-ref-def/Fenced 含实体的双因例，全 26 节 0 回归。

## 改两个文件
- **`entry/src/main/ets/parser/commonmark/Inlines.ts`**（主）
- **`entry/src/main/ets/parser/commonmark/Blocks.ts`**（info string 一处）
不动 Renderer / HtmlEntities.ts / Node。

## 背景：现有 dest/title/info 管线（已 Hermes 摸清）
- `parseAngleDest`(约662)/`parseBareDest`(约691/707/722)/`parseTitle`(约750/773) 内部各调 `resolveBackslashEscapes(content/raw)` 终结 url/title。**这两个解析器同时被 inline 链接(`tryParseLinkOrImage`)和 ref-def(`parseLinkRefDefLine`)复用** → 改一处两边都生效。
- `tryParseLinkOrImage`（约610-611）：`let url = resolveBackslashEscapes(urlRaw); url = normalizeUri(url);` —— 这里对**已被解析器 resolve 过的 urlRaw 又 resolve 一次**（冗余双重，本批顺手去掉）。
- ref-def（约1330/1338）：`normalizeUri(angleRes.url / bareRes.url)`——dest 已 normalizeUri；title（约1352）直接取 `titleRes.title`。**ref-def 复用解析器，故改解析器后 ref-def 免费获得实体解码。**
- ref-def **label**（约1312 `resolveBackslashEscapes(rawLabel)`）：**保持不动**（label 不解码实体）。
- code fence info（Blocks.ts 约100）：`node.attrs.info = fence.substring(infoStart).trim()` —— **当前完全无 unescape**，本批加。

## CommonMark 规则
link/image 的 dest、title，以及 code fence 的 info string，都要做 `unescapeString` = **反斜杠转义 + 实体引用（命名+数字）单趟解码**。dest 解码后再 normalizeUri（顺序：`unescapeString` → `normalizeUri`）。
- 单趟很重要：`\&amp;` 应是反斜杠转义产出字面 `&` + 文本 `amp;` = `&amp;`（实体**不**再解码）；分两趟（先全反斜杠再全实体）会把它错解成 `&`。复用 `tryParseEntity` 在同一趟里处理 `&...;`。

## 目标实现

### Inlines.ts

**1) 新增 `export function unescapeString`（单趟，复用 isEscapable + tryParseEntity）**
```ts
/** 单趟反斜杠转义 + 实体解码（用于 dest/title/info）。
 *  \X(X 为 ASCII 标点) → X；&...;(命名/数字实体) → 解码值；其余原样。 */
export function unescapeString(s: string): string {
  let result: string = '';
  let i: number = 0;
  while (i < s.length) {
    const ch: string = s[i];
    if (ch === '\\' && i + 1 < s.length && isEscapable(s[i + 1])) {
      result += s[i + 1];
      i += 2;
    } else if (ch === '&') {
      const er: EntityResult | null = tryParseEntity(s, i);
      if (er !== null) {
        result += er.value;
        i = er.nextIdx;
      } else {
        result += ch;
        i += 1;
      }
    } else {
      result += ch;
      i += 1;
    }
  }
  return result;
}
```
> `tryParseEntity` 与 `isEscapable` 都在本文件，直接调用。`unescapeString` 需 `export`（Blocks.ts 用）。

**2) dest/title 解析器：把 `resolveBackslashEscapes` 换成 `unescapeString`**（共 6 处）
- `parseAngleDest` 约662：`const url: string = unescapeString(content);`
- `parseBareDest` 约691、707、722：三处 `unescapeString(content)`
- `parseTitle` 约750、773：两处 `unescapeString(raw)`

**3) `tryParseLinkOrImage` 去掉冗余二次 resolve**（约610-611）
```ts
// 现状
let url: string = resolveBackslashEscapes(urlRaw);
url = normalizeUri(url);
// 改为（urlRaw 已由解析器 unescapeString 过）
const url: string = normalizeUri(urlRaw);
```
> 注意下方若有对 `url` 的再赋值需保持兼容；若原 `let url` 在后续被复用，保留 `let` 也可，只是删掉 `resolveBackslashEscapes(urlRaw)` 那一次调用、直接 `url = normalizeUri(urlRaw)`。

**4) 不改**：ref-def 的 dest/title（复用解析器，自动生效）、ref-def label（约1312 保持 `resolveBackslashEscapes`）、`resolveBackslashEscapes` 函数本体（其它地方可能仍用，保留）。

### Blocks.ts

**5) code fence info string 加 unescape**
- 顶部 import 增加 `unescapeString`：把现有 `import { parseInlines, parseLinkRefDefLine, parseHtmlOpenTag, parseHtmlCloseTag } from './Inlines';` 加上 `unescapeString`。
- `parseCodeBlock` 约100：
```ts
node.attrs.info = unescapeString(fence.substring(infoStart).trim());
```

## 逐例验证逻辑
- **Ex32** `[foo](/f&ouml;&ouml; "f&ouml;&ouml;")`：dest `/f&ouml;&ouml;`→unescapeString→`/föö`→normalizeUri→`/f%C3%B6%C3%B6`；title `f&ouml;&ouml;`→unescapeString→`föö`。→ `<a href="/f%C3%B6%C3%B6" title="föö">foo</a>`。
- **Ex33** `[foo]\n\n[foo]: /f&ouml;&ouml; "f&ouml;&ouml;"`：ref-def 复用解析器 → 同 Ex32 结果。
- **Ex34** ` ``` f&ouml;&ouml;\nfoo\n``` `：info `f&ouml;&ouml;`→unescapeString→`föö`→`<code class="language-föö">`。
- 连带：任何 dest/title/info 含命名或数字实体的用例（Links/Images/Link-ref-def/Fenced）应一并改善。

## ArkTS 硬约束
- 无正则；`unescapeString` 纯 char-loop + 复用 `tryParseEntity`/`isEscapable`。
- 无匿名对象字面量（返回 string；`er` 是 `EntityResult|null` 具名类）。
- 无 Record / 无 `obj[key]`；字符串 `s[i]` 合法。
- import 在顶部（Blocks.ts 顶部 import 加 `unescapeString`）。
