# Phase 8.2c-2 — 引用式链接 + 链接引用定义（reference links）

接 8.2c-1。实现 CommonMark **引用式链接**：`[text][label]`(full) / `[text][]`(collapsed) / `[label]`(shortcut)，配合**链接引用定义** `[label]: destination "title"`。定义可出现在使用**之后**（前向引用）⇒ 必须**两遍**：先全文收集定义入表，再 inline 解析引用。

## 范围 / 不做
- **做**：单行链接引用定义 `[label]: dest "title"`（label 后紧跟 `:`，同一逻辑行内给出 dest + 可选 title）；三种引用式链接解析；引用式**图片** `![alt][label]` 同理。
- **不做（留后续，别硬上）**：多行定义（label 行 / dest 行 / title 行分跨）；定义里 dest/title 复杂换行；嵌套引用边角。先把单行定义这条主路打通。
- 复用 8.2c-1 已有的 `parseAngleDest` / `parseBareDest` / `parseTitle` / `resolveBackslashEscapes` / `encodeSpacesInUrl`（在 Inlines.ts；需要就 `export` 它们，别重复实现）。

## 架构（ArkTS 严格模式！）
### 1) 新模块 `commonmark/LinkRefs.ts`
```ts
export class LinkRefDef { url: string = ''; title: string = ''; }
class LinkRefStore {
  private map: Map<string, LinkRefDef> = new Map<string, LinkRefDef>();
  clear(): void { this.map.clear(); }
  // label 归一化：trim → 折叠内部连续空白为单个空格 → 转小写。空 label 返回 ''。
  normalize(label: string): string { /* 用 split/join 折叠空白，不要正则 \\ 反斜杠类 */ }
  set(label: string, def: LinkRefDef): void { const k = this.normalize(label); if (k !== '' && !this.map.has(k)) this.map.set(k, def); } // 首个定义优先(CommonMark)
  get(label: string): LinkRefDef | undefined { return this.map.get(this.normalize(label)); }
  has(label: string): boolean { return this.map.has(this.normalize(label)); }
}
export const linkRefs: LinkRefStore = new LinkRefStore();
```
- **必须用 `Map`，禁止 `Record`/索引访问**（ArkTS）。`LinkRefDef` 具名 class，**禁匿名对象字面量**（8.2c-1 踩过的坑）。

### 2) 预扫 + 清表：在 `ToastMark.parse`（ToastMark.ts）
- `parse()` 开头：`linkRefs.clear();` 然后**预扫**整个 markdown 收集定义（在 `parseBlocks` 之前）。预扫逻辑：逐行找 `^ {0,3}\[label\]:` 形式 → 用复用的 dest/title 解析器解析该行 `]:` 之后的内容 → 合法则 `linkRefs.set(label, def)`。预扫**只收集、不改 state 主流程**（可另起一个轻量行迭代，别动 parseBlocks 的 state）。
- 同一 label 多次定义：**保留第一个**（set 里已用 `!has` 守卫）。

### 3) 定义行不渲染：`tryParseBlock`(Blocks.ts)
- 在 paragraph 兜底**之前**加：若当前行是合法单行链接引用定义（同一判定函数 `isLinkRefDefLine(line)`）→ `state.nextLine()` 消费它 → 返回一个**渲染为空的节点**（最简：`new AstNode(AstNodeType.Paragraph)` 不加子节点——renderParagraph 对空内容返回 ''；parseBlocks 对 children>0 才 append，空段落不会输出，但要确保 state 已前进）。
  - ⚠️ 关键：**必须推进 state**，否则死循环（参考 8.2a 的 pipe 死循环教训）。
- 注意优先级：定义判定要在「裸 `[` 当普通段落」之前；但不能误吞普通段落里以 `[` 开头却不是定义的行（无 `]:` dest 的不算定义）。

### 4) inline 解析引用：`parseInlines`(Inlines.ts)，在现有内联链接 `[text](...)` 分支**之后**、且要处理 `[` 开头
现有顺序：图片`![`(62) → 链接`[`(77, 走 tryParseLinkOrImage 内联) → ... → 脚注`[^`(170)。
对 `[`（和 `![`）增加引用式回退：当 `tryParseLinkOrImage`(内联 `(...)`) 失败时，尝试引用式：
- **full** `[text][label]`：`]` 后紧跟 `[label]`。用 `linkRefs.get(label)`；命中 → Link(url/title=def, 子节点 parseInlines(text))；未命中 → 不是链接（文本）。
- **collapsed** `[text][]`：`]` 后紧跟 `[]`。label = text。
- **shortcut** `[text]`：`]` 后**不**跟 `(` 也不跟 `[`。label = text。命中表才成链接，否则普通文本。
- 图片 `![...]` 同理（命中则 Image，alt=text，url/title=def）。
- ⚠️ 不要破坏脚注 `[^label]`（它在 shortcut 之前判定/优先）。shortcut 的 text 里含 `^` 开头的别误判。
- ⚠️ 不要破坏 8.2c-1 的内联链接/图片（先试内联 `(...)`，失败再试引用式）。
