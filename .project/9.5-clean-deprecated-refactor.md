## 1. PhotoViewPicker → photoAccessHelper（`ImageEditor.ets`）

### 1.1 旧代码（当前）
```ts
import { picker } from '@kit.CoreFileKit';   // L10

// L26-30
const photoPicker = new picker.PhotoViewPicker();
const opt = new picker.PhotoSelectOptions();
opt.MIMEType = picker.PhotoViewMIMETypes.IMAGE_TYPE;
opt.maxSelectNumber = 1;
photoPicker.select(opt).then((res: picker.PhotoSelectResult): void => { ... })
```

### 1.2 新 API 迁移（HarmonyOS NEXT API 12）

**import 改**：
```ts
import { photoAccessHelper } from '@kit.MediaLibraryKit';
```

**类名映射**：
```
picker.PhotoViewPicker       → photoAccessHelper.PhotoViewPicker
picker.PhotoSelectOptions    → 直接对象字面量（select() 参数）
picker.PhotoViewMIMETypes    → photoAccessHelper.PhotoViewMIMETypes
picker.PhotoSelectResult     → photoAccessHelper.PhotoSelectResult (或编译器推断)
```

**重构后的 `pickFromGallery()` 方法**（L23-50）：
```ts
private pickFromGallery(): void {
  const self: ImageEditor = this;
  try {
    const photoPicker = new photoAccessHelper.PhotoViewPicker();
    photoPicker.select({
      MIMEType: photoAccessHelper.PhotoViewMIMETypes.IMAGE_TYPE,
      maxSelectNumber: 1
    }).then((res: photoAccessHelper.PhotoSelectResult): void => {
      try {
        if (res && res.photoUris && res.photoUris.length > 0) {
          const uri: string = res.photoUris[0];
          const imageDir: string = self.getUIContext().getHostContext()!.filesDir + '/userimg';
          const name: string = fileService.copyImageToSandbox(uri, imageDir);
          self.imageUrl = 'userimg/' + name;
          self.pickStatus = '已选择: ' + name;
        } else {
          self.pickStatus = '未选择图片';
        }
      } catch (e) {
        self.pickStatus = '复制失败: ' + ((e as Error).message !== undefined ? (e as Error).message : 'unknown');
      }
    }).catch((e: Object): void => {
      self.pickStatus = '选择出错: ' + String(e);
    });
  } catch (e) {
    self.pickStatus = '启动选择器出错: ' + ((e as Error).message !== undefined ? (e as Error).message : 'unknown');
  }
}
```
> **要点**：
> - 去掉 `new picker.PhotoSelectOptions()` 的中间对象（ArkTS 严格模式禁不必要的新建类实例；select 的参数直接用接口对象字面量 `{ MIMEType, maxSelectNumber }`，有明确接口类型 → 允许）。
> - 类型注解 `res: photoAccessHelper.PhotoSelectResult`——若编译报该类型不存在，改为 `res: Object` 或省略注解（只保留参数名）。
> - 其余连带的 `photoUris`/`.select` 用法不变。

### 1.3 如编译报错的对策
- `@kit.MediaLibraryKit` 不存在 → 试 `@ohos.file.photoAccessHelper`（直接 import photoAccessHelper）。
- `photoAccessHelper.PhotoViewMIMETypes` 不存在 → 试字符串 `'image/*'` 作为 MIMEType（部分版本支持字符串）。
- `photoAccessHelper.PhotoSelectResult` 类型不存在 → 去除类型注解或改为 `Object`。

## 2. showToast → openToast（`EditorPage.ets` L343）

### 2.1 旧代码
```ts
// L340-348
private showToast(msg: string): void {
  try {
    promptAction.showToast({
      message: msg,
      duration: 2000
    });
  } catch (_e) {}
}
```
> import: `import { promptAction } from '@kit.ArkUI';`（现存）

### 2.2 新 API
```ts
private showToast(msg: string): void {
  try {
    promptAction.openToast({
      message: msg,
      duration: 2000
    });
  } catch (_e) {}
}
```
> import 不变（`promptAction` 已在 `@kit.ArkUI`）。

### 2.3 如编译报错
- `promptAction.openToast` 不存在 → 保留 `promptAction.showToast`（此 API 在部分 SDK 仍可用），标注已知。
