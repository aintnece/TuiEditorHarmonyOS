## CommonMark ref-def 多行规则
1. ≤3 前导空格 + `[`。
2. **label**：`[` 到匹配 `]`，**可跨行**（含 `\n`），非空，≤999 字符，内部未转义 `]` 非法。`]` 后必须 `:`。
3. `:` 后 **dest**：跳空白（**可含至多一个换行** → dest 可在下一行）。dest 用 `parseAngleDest`/`parseBareDest`，**dest 本身不跨行**。
4. dest 后 **title**（可选）：跳空白（**可含至多一个换行**）。title 用 `parseTitle`（`"..."`/`'...'`/`(...)`，**可跨行**，但内部不含空行）。
5. **title 校验**：title 与 dest 间须有空白；title 后到行尾只能是空白。若 title 解析失败 / title 后有非空白 → **无 title，定义在 dest 后结束**（不吞后续行）。
6. ref-def **不能跨空行**（空行终止）。
7. 一个连续块可含**多个**定义（Ex217）：解析完一个，从剩余位置继续。

## 实现方向（commonmark.js 风格：拼 block + 基于位置解析 + 换算行数）
新增 `parseLinkRefDefBlock(lines: string[], start: number): LinkRefDefBlockResult | null`（具名 class 含 `defs: LinkRefDefParseResult[]` + `linesConsumed: number`）：
1. 从 `start` 起把**连续非空行**拼成 `block`（`\n` 连接），到空行/EOF 止。
2. 在 `block` 上**基于位置**反复解析 ref-def（复用现有 `parseAngleDest`/`parseBareDest`/`parseTitle`/`resolveBackslashEscapes`/`normalizeUri`——它们是 `(text, pos)` 接口，能跨 `\n`）：每成功一个，记录 def，pos 推进到该 def 末尾（跳到下一行行首继续）；遇到不匹配则停。
3. `linesConsumed` = 已解析 def 覆盖的 block 文本对应的行数（按 `\n` 计数 + 末尾位置换算）。**至少消费 1 个 def 才返回非 null**（否则 null）。
4. 复用注意：`parseBareDest` 须在遇 `\n` 时停（dest 不跨行）；`parseTitle` 须允许 `\n`（title 跨行）。CC 核对现有实现，若不符做最小适配（不破坏单行调用）。

### 两处接入
- **ToastMark 预扫**（已 lines 数组）：遍历改为 `while i<lines.length: r=parseLinkRefDefBlock(lines,i); if r: { for def in r.defs linkRefs.set(...); i += r.linesConsumed } else i++`。
- **Blocks 消费**（ParseState）：tryParseBlock 第 79-83 改为多行消费。ParseState 有 `save()/restore()/currentLine()/nextLine()`——可 peek 多行（save → 累积行 → 解析 → 按 linesConsumed nextLine 多次；不匹配 restore）。或给 ParseState 加只读 `lineAt(offset)`/暴露剩余行（最小改动，CC 定）。消费 N 行后返回空 Paragraph（不渲染）。
- ⚠️ 旧 `parseLinkRefDefLine` 单行版**保留**（其它地方可能引用）或让新块版内部复用其单行逻辑；不破坏现有单行行为。
