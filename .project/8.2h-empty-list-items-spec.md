       out: <ol>\n<li>foo</li>\n<li></li>\n<li>bar</li>\n</ol>

## 现状（`entry/src/main/ets/parser/commonmark/Blocks.ts`）

### `parseBulletMarker`（345-365）
```ts
if (line.length < 2) return null;          // 「-」单独成行 → null（空 marker 不识别）
const ch = line[0];
if (ch !== '-' && ch !== '*' && ch !== '+') return null;
if (line[1] !== ' ' && line[1] !== '\t') return null;   // marker 后须空白+内容 → 「-」「-   」不过
```
### `parseOrderedMarker`（371-403）
```ts

if (line[i+1] !== ' ' && line[i+1] !== '\t') return null;
```
→ 后果：`-`/`*`/`2.`（marker 后 EOL 或仅空白）不被识别为 marker。Ex281 的 `-\n` 因此落进 item 内容、被 `parseParagraph` 当 **setext 下划线**把上一项 foo 变 `<h2>`（actual `<li><h2>foo</h2></li>`）。

### `parseList`（413-555）
- 458 `firstContent = substringByColumn(line, contentCol)`；463 `itemLines.push(firstContent)`。
- 518-524 tight/loose：`itemLines[j] === ''` 任一为空 → loose。**空 marker 的空 firstContent 会误触发 loose**（Ex279 应 tight）。

- ⚠️ **未处理「空内容 marker + 至多一个起始空行」规则**（Ex279 vs Ex280 的关键区别）。

## CommonMark 规则（本批落地）
1. **空 marker = 合法（空）列表项**：marker 字符（`-`/`*`/`+` 或 `数字.`/`数字)`）后是**行尾或仅空白** → 合法 marker，内容偏移 `contentCol = markerWidth + 1`（bullet markerWidth=1→W=2；ordered markerWidth=位数+1→W=+1）。
2. **「至多一个起始空行」**：若某项的 marker 行**无内容**（firstContent 空/纯空白），则：
   - 紧接的下一行**非空**且缩进 ≥ W → 它是本项内容（Ex279 `-   ` → 下一行 `  foo` → 内容 `foo`）。
   - 紧接的下一行**为空**（或 EOF）→ 本项是**空项**，**不**消费该空行，本项终止（Ex280 `-` → 下一空行 → 空项；后续 `  foo` 在 list 结束后成独立段落）。
3. **空内容不算 loose 触发**：空 marker 的空 firstContent 不应使列表 loose（Ex279 须 tight `<li>foo</li>` 不是 `<li><p>foo</p></li>`）。
4. **空项渲染**：空 ListItem（无子节点）→ `<li></li>`。

## 实现要点（CC 自行读全 parseList + parseBlocks 空行流后实现；只改 Blocks.ts）

### A. parseBulletMarker 加空 marker 分支
marker 字符后 **EOL 或 `isAllWhitespaceFrom(line,1)`** → `contentCol = 2`，返回（不再要求 line[1] 是空白+内容）。否则走原有「marker + 空白 + 内容」逻辑。

### B. parseOrderedMarker 加空 marker 分支
分隔符后 **EOL（`markerEndIdx >= line.length`）或 `isAllWhitespaceFrom(line, markerEndIdx)`** → `contentCol = markerWidth + 1`，返回。否则走原逻辑。
（`isAllWhitespaceFrom` 已存在于本文件，可直接用。）

### C. parseList 处理空内容 marker
在取得 firstContent 后判断 `isBlankLine(firstContent)`（空/纯空白 = 空内容 marker）：
- **空内容 marker**：
  - peek 下一行（已 nextLine 过 marker 行）：若 `state.isEnd()` 或 `isBlankLine(currentLine)` → 本项为**空项**（itemLines = []，**不**进消费空行的内容循环，**不**消费那一空行），产出空 ListItem，continue 外层。
  - 否则（下一行非空）→ itemLines 起始为 **空数组**（不把空白 firstContent 推进去），照常进内容收集循环（让下一行按 ≥W 去缩进归属）。
- **非空内容 marker**：照原逻辑 itemLines=[firstContent]。
- tight/loose 判定（518-524）：跳过「空内容 marker 产生的空首行」——简单做法是空内容 marker 时 itemLines 不含那个空串，自然不触发。

### D. 验证 Ex281 setext 不再误触发
`-\n` 现被识别为 marker → parseList 内层循环遇 `-` 行 break（新项），上一项内容仅 `foo` → `parseBlocks('foo')` 不会把下一行 `-` 当 setext（因为 `-` 已被外层当 marker，不进 item 内容）。CC 跑 harness 确认 Ex281 = `<li>foo</li><li></li><li>bar</li>`。
