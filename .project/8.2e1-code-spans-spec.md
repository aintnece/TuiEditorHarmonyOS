# 8.2e-1 — Code spans（行内代码：反引号 run 等长匹配）

## 目标

把 `entry/src/main/ets/parser/commonmark/Inlines.ts` 的行内代码分支从**朴素单反引号 `indexOf`** 升级为 **CommonMark 0.31.2 标准行内代码算法**（反引号 run 等长匹配 + 换行转空格 + 首尾单空格剥离 + 无闭合则开头 run 留字面）。

## 二、CommonMark 行内代码规则（要实现的行为）

1. **开标记** = 当前位置起连续反引号 run，长度记 `openLen`。
2. **闭标记** = 之后**第一个长度恰好等于 `openLen` 的反引号 run**（长度 ≠ openLen 的 run 跳过，不算闭合）。
3. **找到闭合**：内容 = 开 run 之后、闭 run 之前的原文，然后：
   - (a) **所有换行符转成单个空格**（`\n` 与 `\r` 各转成一个 `' '`；不折叠、不去重）。
   - (b) 若内容**长度>0 且含至少一个非空格字符 且 首字符是空格 且 末字符是空格** → 去掉**一个**首空格 + **一个**末空格。
   - 用处理后的内容建 `AstNodeType.Code` 节点，`i` 推进到**闭 run 之后**。
4. **未找到闭合**：开 run **留作普通文本**（产出一个 Text 节点，内容 = 那串反引号），`i` 仅推进到**开 run 之后**（其后的反引号 run 会在后续扫描里被重新处理——Ex349 靠此：`` `foo``bar`` `` 开头单 `` ` `` 无闭合留字面，后面 `` `` `` 再开成 `<code>bar</code>`）。

> 只有 **ASCII 空格 U+0020** 参与首尾剥离判断（对标 commonmark `contents[0] == " "`），别把 tab/其它空白算进去。

## 三、替换后的分支（照此写）

```typescript
// ── 行内代码 `code`（CommonMark 反引号 run 等长匹配）──
if (text[i] === '`') {
  // 1) 开 run 长度
  let p: number = i;
  while (p < len && text[p] === '`') { p++; }
  const openLen: number = p - i;
  const contentStart: number = p;

  // 2) 找等长闭 run
  let q: number = p;
  let closeStart: number = -1;
  while (q < len) {
    if (text[q] === '`') {
      let r: number = q;
      while (r < len && text[r] === '`') { r++; }
      const runLen: number = r - q;
      if (runLen === openLen) { closeStart = q; break; }
      q = r;                       // 长度不符，跳过整个 run
    } else {
      q++;
    }
  }

  if (closeStart >= 0) {
    // 3) 内容处理：换行→空格 + 首尾单空格剥离
    let content: string = codeSpanNewlinesToSpaces(text.substring(contentStart, closeStart));
    if (content.length > 0 && codeSpanHasNonSpace(content) &&
        content[0] === ' ' && content[content.length - 1] === ' ') {
      content = content.substring(1, content.length - 1);
    }
    const node: AstNode = new AstNode(AstNodeType.Code);
    node.text = content;
    out.push(node);
    i = closeStart + openLen;
    continue;
  } else {
    // 4) 无闭合：开 run 留字面，仅推进过开 run
    const lit: AstNode = new AstNode(AstNodeType.Text);
    lit.text = text.substring(i, p);
    out.push(lit);
    i = p;
    continue;
  }
}
```

## 四、新增 2 个纯函数 helper（放到文件「辅助类型与函数」区，跟现有 helper 一起）

```typescript
/** 行内代码内容：换行符（\n / \r）各转成一个空格（不折叠） */
function codeSpanNewlinesToSpaces(s: string): string {
  let out: string = '';
  for (let k: number = 0; k < s.length; k++) {
    const ch: string = s[k];
    if (ch === '\n' || ch === '\r') { out += ' '; }
    else { out += ch; }
  }
  return out;
}

/** 是否含至少一个非空格(U+0020)字符 */
function codeSpanHasNonSpace(s: string): boolean {
  for (let k: number = 0; k < s.length; k++) {
    if (s[k] !== ' ') return true;
  }
  return false;
}
```

> 命名避免与现有 helper 冲突（grep 确认 `codeSpanNewlinesToSpaces`/`codeSpanHasNonSpace` 当前不存在）。

## 六、必过样例（实现后必须转正）

- Ex329 `` `` foo ` bar `` `` → `<p><code>foo \` bar</code></p>`（N=2，内部单 `` ` `` 不闭合，首尾各剥一空格）
- Ex330 `` ` `` ` `` → `<p><code>``</code></p>`（N=1，`` `` `` 是长度 2 的 run 不闭合，末单 `` ` `` 闭合；内容 ` `` ` 剥首尾空格）
- Ex331 `` `  ``  ` `` → `<p><code> `` </code></p>`（内容 `  ``  ` 剥一首一末空格 → ` `` `）
- Ex335 多行 → 换行转空格：`<p><code>foo bar   baz</code></p>`
- Ex336 `` ``\nfoo \n`` `` → `<p><code>foo </code></p>`
- Ex337 `` `foo   bar \nbaz` `` → `<p><code>foo   bar  baz</code></p>`
- Ex339 `` ``foo`bar`` `` → `<p><code>foo\`bar</code></p>`
- Ex340 `` ` foo `` bar ` `` → `<p><code>foo `` bar</code></p>`
- Ex349 `` `foo``bar`` `` → `<p>\`foo<code>bar</code></p>`（开头单 `` ` `` 无闭合留字面）
- 连带 Hard line breaks：Ex640 `` `code  \nspan` `` → `<p><code>code   span</code></p>`；Ex641 `` `code\\\nspan` `` → `<p><code>code\\ span</code></p>`（code span 内反斜杠是字面、换行转空格）
