## 核心根因（`entry/src/main/ets/parser/commonmark/Blocks.ts`）
- `parseBulletMarker`（345）：`const ch = line[0]` —— **从 line[0] 读，0 前导空格容忍**。` - foo` → null → 落段落。
- `parseOrderedMarker`（371）：数字从 `i=0` 起 —— 同样不容忍前导空格。`  10.` → null。
- 两函数的 `contentCol` 不含前导空格（非绝对列）。
- `isNonEmptyListMarker`（8.2h 新增，parseParagraph/isLazyContinuable 用）：未限制「ordered 打断段落需 start=1」→ Ex304 `14.` 误打断段落。

## CommonMark 规则（本批落地）
1. **marker 前导缩进 0-3**：marker 前可有 0-3 个**空格**（本批不处理 Tab）；≥4 空格 → indented code（非 marker，入口 line 53 已处理，parseXxxMarker 对 ≥4 返回 null）。
2. **contentCol = 绝对列**：`= 前导空格数 N + markerWidth + 后空白W规则`。
   - bullet markerWidth = 1；ordered markerWidth = 位数 + 1（分隔符）。
   - 后空白 s：1≤s≤4 → contentCol = N + markerWidth + s；s≥5 或空 marker → contentCol = N + markerWidth + 1。
3. **后续块按 contentCol 绝对列归属**：现有 parseList 的 `substringByColumn`/`stripIndentCols`/`countIndentColumns` **本就是绝对列语义** → contentCol 改绝对列后，归属/嵌套判定（`indentCols >= contentCol`）自动正确。
4. **段落打断限制**（关键，防 Ex304 回归）：list item 打断段落/懒续行的条件——**bullet 无限制；ordered 仅当 start=1**（且非空，8.2h 已有非空判定）。

## 实现要点（只改 Blocks.ts；CC 读全 parseList 确认绝对列兼容后实现）

### A. parseBulletMarker 加前导空格
开头数前导**空格**（非 tab）：`let p=0; while(p<line.length && line[p]===' ' && p<4) p++;`。若 `p>=4` return null（indented code）。marker 字符在 `line[p]`（非 line[0]）。空 marker 分支（8.2h）：`line[p]` 后 EOL/全空白 → contentCol = p + 1 + 1。非空：`line[p+1]` 须空白；wsCols 从 p+1 算；contentCol = p + 1 + (1≤wsCols≤4 ? wsCols : 1)。**注意 countWhitespaceFrom 的 startCol 参数要传 p+1（marker 字符占第 p 列，其后从列 p+1 起）**。

### B. parseOrderedMarker 加前导空格
开头同样数前导空格 p（≤3，≥4 null）。数字从 `line[p]` 起。markerWidth = p + 位数 + 1。空 marker / 非空逻辑同理，contentCol 含 p。

### C. 段落打断 start=1（防 Ex304）
`isNonEmptyListMarker`（或新拆分逻辑）：bullet 非空 marker → 可打断（返回 true）；ordered 非空 marker → **仅 num===1 才返回 true**（start≠1 不打断段落）。
⚠️ 注意：此限制**只作用于 parseParagraph + isLazyContinuable 的「打断段落」判定**；tryParseBlock 入口（顶层启动 list）和 parseList 内容循环 break **不受此限**（顶层 `14.` 仍能启动 list；只是不能打断已有段落）。

### D. parseList 兼容性核对（CC 必做）
- 首行 marker 解析（422-434）+ 外层循环 marker 解析（443-453）→ contentCol 现为绝对列。
- 首行内容 `substringByColumn(line, contentCol)`（458）→ 绝对列，正确剥前导空格+marker。
- 内层归属（478-503）：`countIndentColumns(next) >= contentCol`（绝对列比较）→ 嵌套内容；`< contentCol` + 是 marker → break（同级/新 list）。**确认现有逻辑在绝对列下对 Ex295/310/312 的同级 vs 嵌套判定正确**。
- 若发现需调整 parseList 主体，最小改动 + 在报告里说明（属「marker 识别变化连带」）。
