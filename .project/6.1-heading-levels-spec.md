## 实现要求

把 `Toolbar.ets` 第 106 行那一行固定 H2 的标题按钮，替换成一个带下拉菜单的标题按钮。

### 按钮视觉（必须与现有 iconBtn 完全一致）

- 用 `$r('app.media.tui_heading')` 图标，Button `ButtonType.Normal`，32×32，padding 0，Image 16×16。
- hover 背景：`this.hoveredTip === '标题' ? this.themeColors.toolbarHover : Color.Transparent`，borderRadius 4，`.margin({ left: 2, right: 2 })`。
- `.onHover` 同 iconBtn：进入设 `this.hoveredTip = '标题'`，离开且当前是 '标题' 则清空。
- 保留 hover tooltip：`.bindPopup(this.hoveredTip === '标题', { message: '标题', placement: Placement.Bottom, mask: false })`。

> 由于要同时挂 `.bindMenu`（点击触发）和 `.bindPopup`（hover 触发的 tooltip），通用 `@Builder iconBtn` 不再适用（它的 onClick 直接执行 action）。请**新建一个专用 `@Builder headingBtn()`**（结构照搬 iconBtn 视觉，但把 `.onClick(action)` 去掉、改挂 `.bindMenu(...)`），在 build() 里用 `this.headingBtn()` 替换原第 106 行那一行。

### 下拉菜单（点击按钮弹出）

- 用 `.bindMenu(this.headingMenu)` 挂一个 `@Builder headingMenu()`。
- 菜单内容：6 个 `MenuItem`，分别对应 H1–H6。
  - content 建议：`标题 1`…`标题 6`（可在 content 里带上 `H1`…`H6` 副标识，按官方 MenuItem 示例的合法写法来；以清晰、与编辑器风格协调为准）。
  - 每个 `MenuItem` 的 `.onClick(() => { if (editorContext.editor) { editorContext.editor.exec('Heading', '<N>'); } })`，N 为对应的 '1'…'6' 字符串。
- 不要加「正文/段落（取消标题）」选项——本批次只做 H1–H6，留待后续。

### ArkTS 严格模式注意

- 回调里引用的是全局 `editorContext.editor`（非 `this.xxx`），无需 `const self = this`。
- `Menu` / `MenuItem` 的参数用法**严格照官方 ComponentUXExamples 的 .ets 写**（MenuItem 的选项对象、bindMenu 的参数形式都以官方示例为准），避免 `arkts-no-untyped-obj-literals`。
- 不用展开运算符、不用 Record、不用索引访问。
