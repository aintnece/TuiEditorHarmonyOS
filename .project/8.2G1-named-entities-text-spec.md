# Phase 8.2-G1 — 命名实体表 + 文本解码（CommonMark 2.5 Entity references）

## 目标
把行内文本里的 **HTML5 命名实体引用**（`&copy;`/`&AElig;`/`&ngE;` 等，2125 个 `;` 结尾项）真正解码。当前 `tryParseEntity` 命名分支只硬编码 6 个。基线 exact **81.60% (532/652)**，Entity 节当前 13/17（失败 25/32/33/34）。本批只做 **文本上下文解码（Ex25）**；dest/title/info 上下文（Ex32/33/34，碰 normalizeUri 管线）留 **G-2** 单独做。目标：Entity 节 +Ex25、全 26 节 0 回归。


- name 不含 `&`/`;`，**大小写敏感**（`AElig` ≠ `aelig`）。
- 已含旧的 6 个（amp/lt/gt/quot/apos/nbsp），且 `nbsp=160`（真 NBSP，修正旧代码错映射成普通空格 32）。

## 目标实现（Inlines.ts）

### import（文件顶部，与现有 import 一起）
```ts
import { HTML_ENTITY_PACKED } from './HtmlEntities';
```

### 新增 helper：惰性建表 + 查找
```ts
/** 命名实体表（惰性构建，name → 解码字符串）。 */
let namedEntityMap: Map<string, string> | null = null;

/** 把 HTML_ENTITY_PACKED ("name=cp,cp;...") 解析进 Map。仅首次调用时构建。 */
function getNamedEntityMap(): Map<string, string> {
  if (namedEntityMap !== null) return namedEntityMap;
  const m: Map<string, string> = new Map<string, string>();
  const entries: string[] = HTML_ENTITY_PACKED.split(';');
  for (let i: number = 0; i < entries.length; i++) {
    const entry: string = entries[i];
    const eq: number = entry.indexOf('=');
    if (eq <= 0) continue;
    const name: string = entry.substring(0, eq);
    const cpsPart: string = entry.substring(eq + 1);
    const cpStrs: string[] = cpsPart.split(',');
    let decoded: string = '';
    for (let j: number = 0; j < cpStrs.length; j++) {
      const cp: number = parseInt(cpStrs[j], 10);
      decoded += String.fromCodePoint(cp);     // 逐个拼，别用 spread（arkts-no-spread）
    }
    m.set(name, decoded);
  }
  namedEntityMap = m;
  return m;
}

/** 查命名实体（name 不含 & 和 ;）。命中返回解码字符串，否则 null。 */
function lookupNamedEntity(name: string): string | null {
  const m: Map<string, string> = getNamedEntityMap();
  const v: string | undefined = m.get(name);
  return v === undefined ? null : v;
}
```

### 替换命名分支为表查找
```ts
  // ── 命名实体（HTML5 全表，2125 项）──
  const semi: number = text.indexOf(';', start);
  if (semi < 0 || semi - start - 1 < 1 || semi - start - 1 > 32) return null;   // 名长 1..32
  const name: string = text.substring(start + 1, semi);   // & 与 ; 之间
  const decoded: string | null = lookupNamedEntity(name);
  if (decoded === null) return null;
  const er2: EntityResult = new EntityResult();
  er2.value = decoded;
  er2.nextIdx = semi + 1;
  return er2;
```
> 名长上界 32（最长实体名 `CounterClockwiseContourIntegral` = 31）。`semi - start - 1` = name 长度。

## 逐例验证逻辑
- **Ex25** `&nbsp; &amp; &copy; &AElig; &Dcaron;\n&frac34; &HilbertSpace; &DifferentialD;\n&ClockwiseContourIntegral; &ngE;`：
  - `&nbsp;`→U+00A0、`&amp;`→`&`(渲染再转义回 `&amp;`)、`&copy;`→©、`&AElig;`→Æ、`&Dcaron;`→Ď、`&frac34;`→¾、`&HilbertSpace;`→ℋ、`&DifferentialD;`→ⅆ、`&ClockwiseContourIntegral;`→∲、`&ngE;`→≧̸(2 码点)。
  - 期望 `<p>  &amp; © Æ Ď\n¾ ℋ ⅆ\n∲ ≧̸</p>`（开头是 NBSP+空格）。
- 未知实体名（如 `&foo;`）→ lookup 返回 null → 原样保留（当作普通文本，由后续 `&` 渲染成 `&amp;`），与现状一致。
- **Ex32/33/34（不修）**：链接 dest/title（`[foo](/f&ouml;&ouml; "f&ouml;&ouml;")`）和 code fence info string 的实体解码留 G-2（需在 dest/title/info 提取后跑 unescape 且 dest 解码后接 normalizeUri）。本批**确认它们不回归**（仍 FAIL）。

## ArkTS 硬约束
- **`Map<string,string>` 允许**（替代 Record）；`m.get`/`m.set` 合法。
- **不用 spread**（`String.fromCodePoint(...arr)` 禁）——多码点用 `for` 循环逐个 `+=`。
- 无正则；用 `split`/`indexOf`/`substring`/`parseInt`/`String.fromCodePoint`。
- 无匿名对象字面量（返回 `string|null` / `Map` / `number`）。
- 无 `obj[key]` 对象属性下标；数组下标 `entries[i]`/`cpStrs[j]`、字符串 `text[start]` 合法。
- import 在顶部。
- 惰性 Map 用模块级 `let namedEntityMap: Map<string,string> | null = null`（ArkTS 允许模块级可变 let + Map）。
