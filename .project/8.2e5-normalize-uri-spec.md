## 一、CommonMark normalizeURI（mdurl.encode）规则

逐**码点**扫描 URL 字符串，输出：
- **保留**：ASCII 字母数字（`A-Za-z0-9`）。
- **保留**：安全标点集 `; / ? : @ & = + $ , - _ . ! ~ * ' ( ) #`。
- **保留已转义**（keepEscaped）：`%` 后紧跟两个十六进制位 → 原样保留 `%XX`（如已有的 `%20`）。
- **其余全部百分号编码**（按 **UTF-8 字节**，每字节 `%XX` 大写十六进制）：空格→`%20`、`"`→`%22`、`\`→`%5C`、`[`→`%5B`、`]`→`%5D`、`<`→`%3C`、`>`→`%3E`、NBSP(U+00A0)→`%C2%A0`、`ä`→`%C3%A4`、`φ`→`%CF%86`、astral（如 emoji）→ 4 字节。
- 孤立 `%`（后面非两 hex）→ 编码成 `%25`。

> `&` 在安全集 → 保留为 `&`，再由 renderer HTML-escape 成 `&amp;`（与现状一致，不会变 `%26`）。

## 二、新增 helper + normalizeUri（放到 helper 区，靠近 isHexDigit/isAsciiAlnum）

```typescript
/** URL 安全标点：; / ? : @ & = + $ , - _ . ! ~ * ' ( ) # */
function isUriSafePunct(c: number): boolean {
  return c === 59 || c === 47 || c === 63 || c === 58 || c === 64 || c === 38 ||
         c === 61 || c === 43 || c === 36 || c === 44 || c === 45 || c === 95 ||
         c === 46 || c === 33 || c === 126 || c === 42 || c === 39 || c === 40 ||
         c === 41 || c === 35;
}

/** 一个字节 → "%XX"（大写十六进制） */
function uriHexByte(b: number): string {
  const HEX: string = '0123456789ABCDEF';
  return '%' + HEX[(b >> 4) & 0xF] + HEX[b & 0xF];
}

/** 码点 → UTF-8 字节的百分号编码 */
function pctEncodeCodePoint(cp: number): string {
  let s: string = '';
  if (cp < 0x80) {
    s += uriHexByte(cp);
  } else if (cp < 0x800) {
    s += uriHexByte(0xC0 | (cp >> 6));
    s += uriHexByte(0x80 | (cp & 0x3F));
  } else if (cp < 0x10000) {
    s += uriHexByte(0xE0 | (cp >> 12));
    s += uriHexByte(0x80 | ((cp >> 6) & 0x3F));
    s += uriHexByte(0x80 | (cp & 0x3F));
  } else {
    s += uriHexByte(0xF0 | (cp >> 18));
    s += uriHexByte(0x80 | ((cp >> 12) & 0x3F));
    s += uriHexByte(0x80 | ((cp >> 6) & 0x3F));
    s += uriHexByte(0x80 | (cp & 0x3F));
  }
  return s;
}

/** CommonMark normalizeURI：保留 alnum/安全标点/已有 %XX，其余按 UTF-8 百分号编码 */
function normalizeUri(s: string): string {
  let result: string = '';
  let i: number = 0;
  const n: number = s.length;
  while (i < n) {
    const c0: number = s.charCodeAt(i);
    if (isAsciiAlnum(c0)) { result += s[i]; i++; continue; }
    if (isUriSafePunct(c0)) { result += s[i]; i++; continue; }
    // keepEscaped：% + 两位 hex 原样保留
    if (c0 === 37 && i + 2 < n && isHexDigit(s[i + 1]) && isHexDigit(s[i + 2])) {
      result += s.substring(i, i + 3);
      i += 3;
      continue;
    }
    // 百分号编码（含 surrogate pair → astral 码点）
    let cp: number = c0;
    let adv: number = 1;
    if (c0 >= 0xD800 && c0 <= 0xDBFF && i + 1 < n) {
      const c1: number = s.charCodeAt(i + 1);
      if (c1 >= 0xDC00 && c1 <= 0xDFFF) {
        cp = 0x10000 + ((c0 - 0xD800) << 10) + (c1 - 0xDC00);
        adv = 2;
      }
    }
    result += pctEncodeCodePoint(cp);
    i += adv;
  }
  return result;
}
```

> `isAsciiAlnum`、`isHexDigit` 已存在，直接用。`s[i]` 取字符、`s.charCodeAt(i)`、`HEX[idx]`(字符串索引) 在本项目合规。

## 六、必过样例（normalizeURI 单独可翻）

- Ex502 `[link](foo\bar)` → `<a href="foo%5Cbar">link</a>`（`\b` 非转义，`\`→`%5C`）
- Ex504 `[link]("title")` → `<a href="%22title%22">link</a>`（`"`→`%22`）
- Ex507 `[link](/url␠"title")`（␠=NBSP U+00A0）→ `<a href="/url%C2%A0%22title%22">link</a>`
- Ex603 `<https://example.com/\[\>` → `<a href="https://example.com/%5C%5B%5C">https://example.com/\[\</a>`
- （若存在）希腊字母 URL `αγω` 类 → `/%CF%86%CE%BF%CF%85` 等非 ASCII 编码

> ⚠️ 预期**不翻**（双因，需命名实体表，不在本批，只要不回归）：Ex503 `foo%20b&auml;`（`&auml;`→`ä` 需表，且 `%20` 必须 keepEscaped 保留）、Ex32/33（`&ouml;` 需表）。
> ⚠️ **不得回归**已通过的 Links(62)/Images(11)/Autolinks(18)/Link-ref-def——normalizeURI 对纯 ASCII 安全 URL 输出不变，`&`/`%XX` 保留语义不变。
