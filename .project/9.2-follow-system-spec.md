## 0. 用户已拍板的取舍
- **交互**：标题栏现有主题按钮改 **bindMenu 三选一**（浅色/深色/跟随系统），当前项打勾。参照已有 H1-H6 标题按钮的 bindMenu 写法（Toolbar，6.1 真机过）。
- **默认值**：首次/无持久化 = **跟随系统**。

## 1. 现状链路（已定位）
- 切换入口：`EditorPage.ets` L470-487 主题 Button → `editor.toggleTheme()`（二态）。
- `Editor.toggleTheme()`(L132) → `EditorCore.toggleTheme()`(L184)：① `themeService.toggleTheme()` ② **重建 `this.renderer = new HtmlRenderer({darkMode})`** ③ `emit('stateChange')`。
- 两条通知并行：`ThemeService.notifyListeners` → `EditorPage.onThemeChange`(L413, 更新外壳 this.theme)；`EditorCore` 的 `emit stateChange` → MdPreview/WwEditor 各自比较 isDark 变化后重载/`setTheme`。
- `ThemeService`：`ThemeMode` 仅 Light/Dark；`init(context)` 读 prefs(PREF_KEY='theme_mode') → setTheme；`savePreference` 存 currentTheme.mode。
- `app.ets`(`MainAbility extends UIAbility`)：`onCreate` 调 `ThemeService.getInstance().init(this.context)`；**无 onConfigurationUpdate**；项目零系统亮暗监听。

## 2. 核心架构决策：重建链路改「订阅式」
现「重建 renderer + emit stateChange」是命令式写在 `EditorCore.toggleTheme` 内，**系统自动变化走不到**。改为 **EditorCore 订阅 ThemeService.onThemeChange**，回调统一重建。这样用户手选 / 系统自动切 都走同一条链路，三处（外壳+预览+WYSIWYG）同步。

## 3. ThemeService.ts 改造（`services/ThemeService.ts`）

### 3.1 ThemeMode 加第三态
```ts
export enum ThemeMode {
  Light = 'light',
  Dark = 'dark',
  System = 'system',
}
```
> ⚠️ `ThemeColors` class **不动**（它表示具体调色板，只有 Light/Dark 两套；System 不是一种颜色）。`ThemeColors.mode` 仍只会是 Light 或 Dark。

### 3.2 ThemeService 新增字段
```ts
private userMode: ThemeMode = ThemeMode.System;  // 用户选择(含 system)，持久化
private systemIsDark: boolean = false;           // 系统当前亮暗(由 app.ets 喂)
```

### 3.3 改造/新增方法
- **`init(context)`**：读 prefs，默认 `'system'`；解析成 ThemeMode 存 `this.userMode`；调 `this.applyEffective()`。（解析用 if 比较字符串，不用索引访问。）
  ```ts
  const saved: string = this.prefs.getSync(PREF_KEY, 'system') as string;
  if (saved === 'dark') { this.userMode = ThemeMode.Dark; }
  else if (saved === 'light') { this.userMode = ThemeMode.Light; }
  else { this.userMode = ThemeMode.System; }
  this.applyEffective();
  ```
  catch 分支：`this.userMode = ThemeMode.System; this.applyEffective();`
- **`setMode(mode: ThemeMode)`**（UI 三选一调）：`this.userMode = mode; this.savePreference(); this.applyEffective();`
- **`setSystemColorMode(isDark: boolean)`**（app.ets 调）：
  ```ts
  if (this.systemIsDark === isDark) return;
  this.systemIsDark = isDark;
  if (this.userMode === ThemeMode.System) { this.applyEffective(); }
  ```
- **`getUserMode(): ThemeMode`**：`return this.userMode;`（UI 打勾用）
- **`toggleTheme()`**（保留兼容，改为基于实际 isDark 切到手动 light/dark）：
  ```ts
  const dark: boolean = this.currentTheme.isDark;
  this.setMode(dark ? ThemeMode.Light : ThemeMode.Dark);
  ```
- **私有 `applyEffective()`**（重算实际调色板 + 通知）：
  ```ts
  const dark: boolean = (this.userMode === ThemeMode.System) ? this.systemIsDark : (this.userMode === ThemeMode.Dark);
  this.currentTheme.copyFrom(dark ? DarkTheme : LightTheme);
  this.notifyListeners();
  ```
- **`savePreference()`**：存 `this.userMode`（不再是 currentTheme.mode）：`this.prefs.putSync(PREF_KEY, this.userMode);`
- **删除旧 `setTheme(mode)`** 的对外语义：原 setTheme 只被 init 内部用，现 init 改走 userMode + applyEffective，可删 setTheme（若担心外部引用，保留为 `setTheme(m){ this.setMode(m); }` 别名也可——grep 确认仅 init 用过）。
- **`getThemeMode()`**：原返回 currentTheme.mode（Light/Dark 实际值）。保留不变（其它地方若用）。新增的 `getUserMode()` 才是三选一高亮用。

> ArkTS：全部 if 比较、显式类型、无 Record/索引访问/对象字面量。

## 4. EditorCore.ts 改造（`editor/EditorCore.ts`）

### 4.1 构造函数末尾订阅主题变化（统一重建链路）
在 `registerBuiltinCommands()` / i18n 之后加：
```ts
// 主题变化(手动 setMode 或 系统 colorMode 变) → 重建预览渲染器 + 广播 stateChange
this.themeService.onThemeChange((t: ThemeColors): void => {
  this.renderer = new HtmlRenderer({ darkMode: t.isDark, fontSize: 16 });
  this.eventEmitter.emit('stateChange');
});
```

```ts
toggleTheme(): void {
```

```ts
setMode(mode: ThemeMode): void {

```

## 6. app.ets 改造（`app.ets`）— 系统亮暗监听

> ⚠️ **ArkTS 坑（2026-06-25 编译实证）**：`config.colorMode` 是**可选属性**，类型 = `ConfigurationConstant.ColorMode | undefined`。中间常量类型注解**必须带 `| undefined`**，否则 ArkTS 严格模式报错 `Type 'ColorMode | undefined' is not assignable to type 'ColorMode'`（10605999）。比较 `cm === COLOR_MODE_DARK` 仍合法（undefined→false=浅色，合理默认）。

### 6.1 import 加 ConfigurationConstant + Configuration
```ts
import { UIAbility, Want, AbilityConstant, ConfigurationConstant, Configuration } from '@kit.AbilityKit';
```
（若 `Configuration` 不在 @kit.AbilityKit，用 `import { Configuration } from '@ohos.app.ability.Configuration';`——CC 按 SDK 实际可用导出确认。）

### 6.2 onCreate 末尾喂初始系统亮暗
在 `ThemeService.getInstance().init(this.context);` **之后**加：
```ts
const cm: ConfigurationConstant.ColorMode | undefined = this.context.config.colorMode;
ThemeService.getInstance().setSystemColorMode(cm === ConfigurationConstant.ColorMode.COLOR_MODE_DARK);
```

### 6.3 新增 onConfigurationUpdate 回调
```ts
onConfigurationUpdate(newConfig: Configuration): void {
  const cm: ConfigurationConstant.ColorMode | undefined = newConfig.colorMode;
  ThemeService.getInstance().setSystemColorMode(cm === ConfigurationConstant.ColorMode.COLOR_MODE_DARK);
}
```
> 时序：onCreate 里 init(读 userMode) → setSystemColorMode(真实系统值)，页面加载前 currentTheme 已是终值，无首帧闪烁。

## 7. EditorPage.ets 改造（`pages/EditorPage.ets`）— 主题按钮改 bindMenu 三选一

### 7.1 import 加 ThemeMode
`import { ThemeService, ThemeColors, ThemeMode } from '../services/ThemeService';`

### 7.2 替换 L470-487 主题 Button 为 bindMenu 三选一
按钮文字显示**当前用户模式名**（浅色/深色/跟随系统，中文硬编码，与现状 L472 中文一致，不碰 i18n）。点击弹菜单，当前项前加 `✓ `。参照 6.1 H1-H6 的 bindMenu 写法（MenuElement[] 对象字面量数组是 ArkUI 标准用法，项目 6.1 已用、真机过——本处沿用，不算违规匿名字面量）。

结构示意（CC 按项目实际 bindMenu 风格落地）：
```ts
// 主题切换 → bindMenu 三选一
Button() {
  Text(this.themeModeLabel()).fontSize(12)
}
  .height(28).padding({ left: 8, right: 8 })
  .backgroundColor(this.theme.toolbarHover)
  .borderRadius(4).fontColor(this.theme.toolbarFg)
  .bindMenu([
    { value: this.menuLabel('浅色', ThemeMode.Light),  action: () => this.pickMode(ThemeMode.Light) },
    { value: this.menuLabel('深色', ThemeMode.Dark),   action: () => this.pickMode(ThemeMode.Dark) },
    { value: this.menuLabel('跟随系统', ThemeMode.System), action: () => this.pickMode(ThemeMode.System) },
  ])
```
配套私有方法（加在 struct 内）：
```ts
private themeModeLabel(): string {
  const m: ThemeMode = ThemeService.getInstance().getUserMode();
  if (m === ThemeMode.Dark) return '深色';
  if (m === ThemeMode.System) return '跟随系统';
  return '浅色';
}
private menuLabel(name: string, mode: ThemeMode): string {
  return (ThemeService.getInstance().getUserMode() === mode ? '✓ ' : '   ') + name;
}
private pickMode(mode: ThemeMode): void {
  if (editorContext.editor) { editorContext.editor.setMode(mode); }
  else { ThemeService.getInstance().setMode(mode); }
  this.theme = ThemeService.getInstance().getTheme();
}
```
> `onThemeChange`(L413) 已会更新 this.theme（系统自动变化时外壳也跟着变）；pickMode 里再刷一次双保险。按钮文字 `themeModeLabel()` 依赖 this.theme 变化触发 build 重算（state 驱动）。
