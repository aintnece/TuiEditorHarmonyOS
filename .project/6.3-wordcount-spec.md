# Phase 6.3 — 字数统计强化（StatusBar 加总行数）spec

## 背景（现状已查）

`components/StatusBar.ets` 已实时显示：`字符`(charCount = markdown.length)、`词数`(countWords，已处理中文字+英文词)、模式标签、`行 N, 列 N`(光标位置)。监听 `change`/`caretChange`/`stateChange`，Markdown / WYSIWYG 两模式都更新（WYSIWYG 经桥 onChange 写回 core → change 事件）。

## 目标

**仅补一项：文档总行数（行数 N）**，实时更新。其余（字符/词数）已就绪不动。

## 唯一改动文件：`components/StatusBar.ets`

### 1. 加状态字段
在现有 `@State private wordCount` 等旁边加：
```
@State private lineCount: number = 1;
```

### 2. 加统计方法（不用正则，用 split）
仿 countWords 风格加一个私有方法：
```
private countLines(text: string): number {
  if (text.length === 0) { return 1; }
  return text.split('\n').length;
}
```
（split('\n') 是普通字符串分割，非正则，符合项目规则。空文档算 1 行。）

### 3. 在两处更新点同步 lineCount
- `aboutToAppear()` 初始值区（现有 `self.charCount = ...; self.wordCount = ...;` 之后）加：
  ```
  self.lineCount = self.countLines(core.getMarkdown());
  ```
- `changeHandler`（现有 `self.charCount = md.length; self.wordCount = self.countWords(md);` 之后）加：
  ```
  self.lineCount = self.countLines(md);
  ```

### 4. build() 加显示
在「词数」那个 Text 之后、`Blank()` 之前，加一个同款 Text：
```
Text('行数 ' + this.lineCount.toString())
  .fontSize(11)
  .fontColor(this.themeColors.editorPlaceholder)
  .margin({ right: 12 })
```

## 约束

- 只改 `components/StatusBar.ets`，不动其它文件。
- ArkTS 严格：用 `const self = this` 已有模式（沿用现有 self）；不用正则、不用对象字面量(SDK 参数除外)、不用 any/Record/索引。
- 样式与现有 字符/词数 Text 完全一致（fontSize 11、同色、margin）。

## 验收（CC 报告）

- 改动仅 StatusBar.ets。
- lineCount 字段 + countLines 方法 + 两处更新 + build() 显示 都已加。
- 字符/词数显示不变。

## 真机验证（用户）

- 编辑器里多敲几行 → 底部状态栏「行数」随回车实时增加；删行减少；空文档显示 1。
- Markdown 与 WYSIWYG 两模式都更新。
- 字符 / 词数 / 模式 / 光标行列 仍正常。
