## 1. getContext(this) → this.getUIContext().getHostContext()（6 处）

**替代规则**：在 @Component 方法内，用 `this.getUIContext().getHostContext()` 替换 `getContext(this)`。二者都返回宿主 Context，`resourceManager`/`filesDir`/`config` 等用法不变（UIContext.getHostContext() = UIAbilityContext extends Context）。

### 文件与位置

| 文件 | 行 | 上下文 |
|---|---|---|
| `MdPreview.ets` | 124 | `onInterceptRequest` 回调中，`getContext(this).resourceManager.getRawFileContentSync(rawPath)` |
| `MdPreview.ets` | 152 | 同上，Prism 资源拦截 |
| `MdPreview.ets` | 179 | 用户图片拦截，`getContext(this).filesDir + '/' + rel` |
| `WwEditor.ets` | 123 | `onInterceptRequest` 回调中，`getContext(this).resourceManager.getRawFileContentSync(rawPath)` |
| `WwEditor.ets` | 139 | 用户图片拦截，`getContext(this).filesDir + '/' + rel` |
| `ImageEditor.ets` | 34 | 图片处理（具体用法需读文件后改） |

### 替换模板
```ts
// 旧
const ctx = getContext(this);
ctx.resourceManager.getRawFileContentSync(...);
// 或内联
getContext(this).filesDir
```
↓
```ts
// 新
const ctx: common.Context = this.getUIContext().getHostContext();
ctx.resourceManager.getRawFileContentSync(...);
// 或内联
this.getUIContext().getHostContext().filesDir
```

> ⚠️ **ArkTS**：若 `const ctx = getContext(this)` 作为单独的 const，替换后需加类型注解 `: common.Context`（或保留无类型让编译器推断，但严格模式建议注解）。如果原代码只是内联使用（未赋给变量），直接链式替换即可。读每个文件后按实际情况改。

## 2. showToast → promptAction.showToast（1 处）

### 文件：`EditorPage.ets` L343

**替代**：
```ts
// 旧
import prompt from '@ohos.prompt';
// ...
prompt.showToast({ message: ..., duration: 2000 });

// 新
import { promptAction } from '@kit.ArkUI';
// ...
promptAction.showToast({ message: ..., duration: 2000 });
```
> 检查文件顶部 import 是否有 `prompt`，有则替换 import、无则加 `promptAction` import。`showToast` 参数格式基本兼容（`message`/`duration`），无需改调用处。

## 3. Function may throw 加异常处理（7 处）

### 文件与行
- `FileService.ets`: 219, 220, 222, 224, 225（共 5 处）
- `WwEditor.ets`: 190, 197（共 2 处）

### 做法
在每处报 WARN 的调用外**包 try-catch**，catch 块内用 `hilog.warn` 记录或保持静默（项目除 app.ets 已无 hilog 调用，可在 catch 块内留空或记 hilog）。若无 hilog import，空 catch 即可：
```ts
try {
  // 原调用
} catch (e) {
  // 静默
}
```
> ⚠️ **ArkTS**：try-catch 内变量作用域正常即可。不要改变调用逻辑，仅加 try-catch 包裹。
