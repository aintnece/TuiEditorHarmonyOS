# 8.2b — Emphasis / Strong（CommonMark delimiter-run 算法）

## 目标

把 `entry/src/main/ets/parser/commonmark/Inlines.ts` 里**朴素 `indexOf` 配对**的强调/加粗解析，替换成 **CommonMark 0.31.2 官方 delimiter-run 算法**（left/right-flanking + rule-of-three + `_` 词内限制 + `***` 嵌套）。
当前 Emphasis 节 46/132，目标显著提升且**全节零回归**。

> 算法蓝本 = commonmark.js 0.31.2 `lib/inlines.js` 的 `scanDelims` / `handleDelim` / `processEmphasis` / `removeDelimiter` / `removeDelimitersBetween`。本 spec 已把它逐行翻译成本项目的 AST 结构 + ArkTS 严格模式可编译形态。**按本 spec 实现即可，不要自行发挥。**

## 只改一个文件

- **`entry/src/main/ets/parser/commonmark/Inlines.ts`** —— 仅此一个文件。
- **不要** 新建文件、**不要** 动 `Node.ts` / `Renderer.ts` / `Blocks.ts` / `LinkRefs.ts` / `Gfm.ts`。
- **不要** 引入 bracket（`[`/`]`）delimiter 栈。本项目链接走 `tryParseLinkOrImage` + 对链接内文**递归** `parseInlines`，链接内的强调天然是独立作用域（见下文「为什么不需要 bracket 栈」），照此实现 Ex419/473/474/478/480 会自然通过。

## 一、现状（要替换 / 要保留）

`parseInlines(text, parent)`（约 28–248 行）当前用 `text.indexOf` 朴素配对，缺 flanking 判定 → 把空格/标点旁的零散 `*`/`_` 误判成强调。三处相关代码：

1. **粗体 `**` / `__`**（约 125–137 行）—— **删掉，并入新 delimiter 分支**。
2. **删除线 `~~`**（约 139–149 行，GFM）—— **原样保留**（`~` 不是 `*`/`_`，与本次无关；它对内文递归 `parseInlines`，自动获得新强调能力，属增益非回归）。
3. **斜体 `*` / `_`**（约 151–172 行）—— **删掉，并入新 delimiter 分支**。

其余分支（转义 `\`、硬/软换行、图片 `![`、链接 `[`、行内代码 `` ` ``、自动链接 `<>`、脚注 `[^`、实体 `&`、普通文本）**逻辑保持不变**，只把它们「`parent.appendChild(x)`」改成「推入本地数组 `out`」（见第二节）。

## 二、整体改造：parseInlines 改成「先收集到 out 数组 + delimiter 栈，最后 processEmphasis」

commonmark 的强调是**两阶段**：扫描时把 `*`/`_` 运行(run)入 delimiter 栈、其它内联照常产出节点；扫描结束后 `processEmphasis` 回头配对、把区间包进 `<em>`/`<strong>`。

本项目 `AstNode` 是**数组子节点模型**（`children: AstNode[]` + `appendChild`，**没有** commonmark 的 `next/prev/unlink/insertAfter` 链表）。所以：
- **内联兄弟节点**用本地数组 `out: AstNode[]` 收集（扫描时 `out.push(node)`，**不要**直接 `parent.appendChild`）。
- **delimiter 栈**用具名 class 双向链表（`previous`/`next` + 顶指针），**逐行照搬** commonmark（最低风险）。
- `processEmphasis` 里 commonmark 的节点链表操作（`unlink`/`insertAfter`/移动区间）**改成对 `out` 数组按对象身份 `indexOf` + `splice`**（数组小，O(n²) 可接受，抗下标漂移）。
- **扫描全部结束后**：`new EmphasisProcessor(out, topDelim).process()` 就地改 `out`；再 `for (const n of out) parent.appendChild(n)` 一次性挂到 parent。

> 递归调用 `parseInlines(innerText, node)`（链接内文、删除线内文）**不变**：每个调用各自建自己的 `out`/delimiter 栈、各自在末尾 `process()` + 挂到自己的 node。互不串扰。

### parseInlines 骨架（改造后）

```typescript
export function parseInlines(text: string, parent: AstNode): void {
  let i: number = 0;
  const len: number = text.length;
  const out: AstNode[] = [];
  let topDelim: Delimiter | null = null;   // delimiter 栈顶

  while (i < len) {
    // ── 转义 \x ──（逻辑不变，改成 out.push）
    //   注意：转义后的 * / _ 作为普通 Text 入 out，绝不进 delimiter 栈（escape 优先级在 * 分支之前）
    // ── 硬/软换行 ──（不变，out.push）
    // ── 图片 ![ ──（不变，out.push）
    // ── 链接 [ ──（不变，out.push；内文仍递归 parseInlines）
    // ── 删除线 ~~ ──（不变，out.push；内文仍递归 parseInlines）  ← 保留！

    // ── 强调/加粗 delimiter：* 或 _ ──（替换原粗体+斜体两块）
    if (text[i] === '*' || text[i] === '_') {
      i = handleDelim(text, i, out, /* 回传新栈顶 */ ...);   // 见第三节
      continue;
    }

    // ── 行内代码 ` ──（不变，out.push）
    // ── 自动链接 <> ──（不变，out.push）
    // ── 脚注 [^ ──（不变，out.push）
    // ── 实体 & ──（不变，out.push）
    // ── 普通文本 ──（不变，out.push 单字符 Text）
  }

  // 扫描结束 → 配对强调 → 挂到 parent
  const proc: EmphasisProcessor = new EmphasisProcessor(out, topDelim);
  proc.process();
  for (let k: number = 0; k < out.length; k++) {
    parent.appendChild(out[k]);
  }
}
```

> **`topDelim` 在主循环里要能被 delimiter 分支更新。** ArkTS 没有方便的 by-ref。两种合规写法任选其一（推荐 A）：
> - **A（推荐）**：不抽 `handleDelim` 函数，把 delimiter 分支逻辑**内联**写在循环里，直接读写局部变量 `topDelim` / `out` / `i`。最简单、无传引用问题。
> - B：把整个扫描循环放进一个 class（字段 `out`/`topDelim`/`pos`），分支调方法。改动更大，不推荐。
>
> 采用 A。下面第三节给出内联版代码。

## 三、delimiter 分支（内联在主循环；替换原 125–137 + 151–172）

```typescript
// ── 强调/加粗：* 或 _ ──
if (text[i] === '*' || text[i] === '_') {
  const cc: string = text[i];
  // 1) 扫描连续同字符 run
  let j: number = i;
  while (j < len && text[j] === cc) { j++; }
  const numdelims: number = j - i;

  // 2) flanking 判定（before/after 用 '\n' 作行首/行尾哨兵，与 commonmark 一致）
  const charBefore: string = (i === 0) ? '\n' : text[i - 1];
  const charAfter: string = (j >= len) ? '\n' : text[j];
  const afterWs: boolean = isUnicodeWhitespace(charAfter);
  const afterPunct: boolean = isPunctuation(charAfter);
  const beforeWs: boolean = isUnicodeWhitespace(charBefore);
  const beforePunct: boolean = isPunctuation(charBefore);

  const leftFlanking: boolean =
    !afterWs && (!afterPunct || beforeWs || beforePunct);
  const rightFlanking: boolean =
    !beforeWs && (!beforePunct || afterWs || afterPunct);

  let canOpen: boolean;
  let canClose: boolean;
  if (cc === '_') {
    canOpen = leftFlanking && (!rightFlanking || beforePunct);
    canClose = rightFlanking && (!leftFlanking || afterPunct);
  } else {  // '*'
    canOpen = leftFlanking;
    canClose = rightFlanking;
  }

  // 3) 产出一个承载整段 run 文本的 Text 节点
  const node: AstNode = new AstNode(AstNodeType.Text);
  node.text = text.substring(i, j);     // 例如 "***"
  out.push(node);

  // 4) 仅当能开或能合时才入栈（既不能开也不能合 → 留作普通文本，如 "**** is not..."）
  if (canOpen || canClose) {
    const d: Delimiter = new Delimiter();
    d.ch = cc;
    d.numdelims = numdelims;
    d.origdelims = numdelims;
    d.node = node;
    d.canOpen = canOpen;
    d.canClose = canClose;
    d.previous = topDelim;
    d.next = null;
    if (topDelim !== null) { topDelim.next = d; }
    topDelim = d;
  }

  i = j;
  continue;
}
```

## 四、新增类型 + 辅助函数（放在文件 250 行附近的「辅助类型与函数」区）

### 4.1 Delimiter（具名 class，禁匿名对象字面量）

```typescript
/** 强调 delimiter 栈节点（双向链表，对标 commonmark.js delimiters） */
class Delimiter {
  ch: string = '';                    // '*' 或 '_'
  numdelims: number = 0;              // 剩余可用 delimiter 数（会被消耗递减）
  origdelims: number = 0;             // 原始 run 长度（rule-of-three 用，不变）
  node: AstNode | null = null;        // 承载 run 文本的 Text 节点
  canOpen: boolean = false;
  canClose: boolean = false;
  previous: Delimiter | null = null;
  next: Delimiter | null = null;
}
```

### 4.2 Unicode 空白 / 标点判定（用 charCode，**不用正则**）

> CommonMark 0.31.2 的标点定义 = ASCII 标点 ∪ Unicode `\p{P}` ∪ Unicode `\p{S}`（symbol，含货币 `£€`）。
> ArkTS 正则 `\p{}` / `\\` 不可靠 → 用下面 charCode 显式实现，已覆盖 spec 全部用例（`$`=ASCII、`£€`=货币符号→标点、西里尔/CJK=字母→非标点）。

```typescript
/** Unicode 空白（含行首/行尾哨兵 '\n'，对标 /^\s/） */
function isUnicodeWhitespace(ch: string): boolean {
  if (ch.length === 0) return true;
  const code: number = ch.charCodeAt(0);
  if (code === 32 || (code >= 9 && code <= 13)) return true;        // space, \t \n \v \f \r
  if (code === 0x00A0 || code === 0x1680) return true;
  if (code >= 0x2000 && code <= 0x200A) return true;
  if (code === 0x2028 || code === 0x2029 || code === 0x202F) return true;
  if (code === 0x205F || code === 0x3000 || code === 0xFEFF) return true;
  return false;
}

/** ASCII 标点码点：33-47 / 58-64 / 91-96 / 123-126 */
function isAsciiPunct(code: number): boolean {
  return (code >= 33 && code <= 47) || (code >= 58 && code <= 64) ||
         (code >= 91 && code <= 96) || (code >= 123 && code <= 126);
}

/** Unicode 标点或符号（CommonMark flanking 用） */
function isPunctuation(ch: string): boolean {
  if (ch.length === 0) return false;
  const code: number = ch.charCodeAt(0);
  if (isAsciiPunct(code)) return true;
  if (code <= 127) return false;                       // 其余 ASCII（字母/数字/控制）非标点
  // 非 ASCII：cased 字母（拉丁/西里尔/希腊等，大小写不同）→ 非标点
  if (ch.toLowerCase() !== ch.toUpperCase()) return false;
  // 无大小写区分的字母（CJK/假名/谚文等，码点 ≥ 0x3040）→ 非标点
  if (code >= 0x3040) return false;
  // 其余非 ASCII（Latin-1 符号、货币 £€、各类 symbol）→ 标点
  return true;
}
```

### 4.3 EmphasisProcessor（逐行翻译 commonmark.js processEmphasis；节点操作改 out 数组 splice）

```typescript
/** 强调/加粗后处理：配对 delimiter，把区间包进 Emph/Strong（对标 commonmark.js processEmphasis(null)） */
class EmphasisProcessor {
  out: AstNode[];
  top: Delimiter | null;   // delimiter 栈顶

  constructor(out: AstNode[], top: Delimiter | null) {
    this.out = out;
    this.top = top;
  }

  /** 链表删除（对标 removeDelimiter） */
  removeDelimiter(d: Delimiter): void {
    if (d.previous !== null) { d.previous.next = d.next; }
    if (d.next === null) {
      this.top = d.previous;            // 删的是栈顶
    } else {
      d.next.previous = d.previous;
    }
  }

  /** 跳过 bottom 与 top 之间的 delimiter（对标 removeDelimitersBetween） */
  removeDelimitersBetween(bottom: Delimiter, top: Delimiter): void {
    if (bottom.next !== top) {
      bottom.next = top;
      top.previous = bottom;
    }
  }

  /** 把 out 中 fromNode 与 toNode 之间(不含两端)的节点包进 wrapper，wrapper 放到 fromNode 之后 */
  private wrapBetween(fromNode: AstNode, toNode: AstNode, wrapper: AstNode): void {
    const oi: number = this.out.indexOf(fromNode);
    const ci: number = this.out.indexOf(toNode);
    // 把 (oi, ci) 之间的节点搬进 wrapper（顺序不变）
    for (let k: number = oi + 1; k < ci; k++) {
      wrapper.appendChild(this.out[k]);
    }
    // 用 wrapper 替换 out[oi+1 .. ci-1]
    this.out.splice(oi + 1, ci - oi - 1, wrapper);
  }

  /** 从 out 移除一个节点（按对象身份） */
  private removeNode(n: AstNode): void {
    const idx: number = this.out.indexOf(n);
    if (idx >= 0) { this.out.splice(idx, 1); }
  }

  process(): void {
    // openers_bottom：14 槽数组（0/1 留给引号，不用；_ 用 2..7，* 用 8..13）
    // 数组下标在 ArkTS 合法（arkts-no-props-by-index 只禁对象属性下标，不禁数组）
    const openersBottom: (Delimiter | null)[] = [];
    for (let n: number = 0; n < 14; n++) { openersBottom.push(null); }
    // stack_bottom = null（整段 parseInlines 一次性处理）

    // 找到栈底之上的第一个 closer：从栈顶回退到底
    let closer: Delimiter | null = this.top;
    while (closer !== null && closer.previous !== null) {
      closer = closer.previous;
    }

    // 前向扫描 closer
    while (closer !== null) {
      const closercc: string = closer.ch;
      if (!closer.canClose) {
        closer = closer.next;
        continue;
      }
      // 找匹配 opener（往回）
      let opener: Delimiter | null = closer.previous;
      let openerFound: boolean = false;
      let obIndex: number;
      if (closercc === '_') {
        obIndex = 2 + (closer.canOpen ? 3 : 0) + (closer.origdelims % 3);
      } else {  // '*'
        obIndex = 8 + (closer.canOpen ? 3 : 0) + (closer.origdelims % 3);
      }
      while (opener !== null && opener !== openersBottom[obIndex]) {
        const oddMatch: boolean =
          (closer.canOpen || opener.canClose) &&
          (closer.origdelims % 3 !== 0) &&
          ((opener.origdelims + closer.origdelims) % 3 === 0);
        if (opener.ch === closer.ch && opener.canOpen && !oddMatch) {
          openerFound = true;
          break;
        }
        opener = opener.previous;
      }
      const oldCloser: Delimiter = closer;

      if (openerFound && opener !== null) {
        // 取实际消耗的 delimiter 数
        const useDelims: number =
          (closer.numdelims >= 2 && opener.numdelims >= 2) ? 2 : 1;
        const openerNode: AstNode | null = opener.node;
        const closerNode: AstNode | null = closer.node;
        opener.numdelims -= useDelims;
        closer.numdelims -= useDelims;
        // 从两端 Text 节点切掉已消耗的 delimiter 字符
        if (openerNode !== null) {
          openerNode.text = openerNode.text.substring(0, openerNode.text.length - useDelims);
        }
        if (closerNode !== null) {
          closerNode.text = closerNode.text.substring(0, closerNode.text.length - useDelims);
        }
        // 建 Emph / Strong，把两端之间节点搬进去
        const emph: AstNode =
          new AstNode(useDelims === 1 ? AstNodeType.Emph : AstNodeType.Strong);
        if (openerNode !== null && closerNode !== null) {
          this.wrapBetween(openerNode, closerNode, emph);
        }
        // 删除 opener 与 closer 之间的所有 delimiter
        this.removeDelimitersBetween(opener, closer);
        // opener 用尽 → 删栈 + 删空 Text 节点
        if (opener.numdelims === 0) {
          if (openerNode !== null) { this.removeNode(openerNode); }
          this.removeDelimiter(opener);
        }
        // closer 用尽 → 删栈 + 删空 Text 节点，closer 前移到 next
        if (closer.numdelims === 0) {
          if (closerNode !== null) { this.removeNode(closerNode); }
          const tempstack: Delimiter | null = closer.next;
          this.removeDelimiter(closer);
          closer = tempstack;
        }
      } else {
        // 没找到 opener：抬高该 closer 类别的搜索下界
        closer = closer.next;
      }

      if (!openerFound) {
        openersBottom[obIndex] = oldCloser.previous;
        if (!oldCloser.canOpen) {
          this.removeDelimiter(oldCloser);
        }
      }
    }
    // out 已就地改好；delimiter 栈丢弃即可（无需清理 this.top）
  }
}
```

> **与 commonmark 的两点等价说明（实现时心里有数即可）：**
> 1. commonmark 把「opener 用尽后 closer 仍继续用」和「没找到 opener 则 `closer=closer.next`」合并在 `* / _` 分支里。上面已等价拆开：找到 opener 时按 useDelims 消耗、closer 没用尽就**留在原地下一轮继续**（不前移）；用尽才前移到 `next`；没找到 opener 才 `closer=closer.next`。**务必照此，别提前 `closer=closer.next`**，否则 `*foo**bar**baz*`(Ex411) 这类一个 closer 配多个 opener 的会断。
> 2. `openersBottom` 用 `null` 作哨兵（= commonmark 的 `stack_bottom=null`）。`opener !== openersBottom[obIndex]` 当槽为 null 时即「搜到栈底为止」。

## 五、为什么不需要 bracket 栈（链接/代码/autolink 交互）

commonmark.js 用 bracket 栈 + `]` 时 `processEmphasis(opener.previousDelimiter)` 把链接内强调限定在 `[...]` 作用域。**本项目用递归实现等价隔离**：

- `tryParseLinkOrImage` 在 `[` 处前向找 `](...)`，对**链接内文**递归 `parseInlines` → 链接内强调是独立 `out`/delimiter 栈，末尾独立 `process()`。外层 delimiter 栈不含链接内的 `*`/`_`（它们在递归里被消费）。
- 这样：
  - **Ex419** `*foo [*bar*](/url)*` → 链接内 `*bar*` 递归成 `<em>bar</em>`；外层 `*foo …*` 配对 → `<em>foo <a><em>bar</em></a></em>` ✓
  - **Ex473** `*[bar*](/url)` → 链接内 `bar*` 的 `*` 无 opener 留字面；外层 `*` 无 closer 留字面 → `*<a>bar*</a>` ✓
  - **Ex478** `*a `*`*` → 行内代码 `` `*` `` 在扫描时先被 code 分支消费（`i` 跳过反引号区间），其中的 `*` 根本不入 delimiter 栈 → 外层 `*a …*` 配对 → `<em>a <code>*</code></em>` ✓
  - **Ex480** `**a<https://foo.bar/?q=**>` → autolink 分支先消费 `<...>`（含内部 `**`）→ 开头 `**` 无 closer 留字面 → `**a<a …>…</a>` ✓
- **因此：保持现有扫描分支顺序与链接/代码/autolink 的「先消费、`i` 跳过」行为不变**，强调只处理那些作为裸文本出现、入了 delimiter 栈的 `*`/`_`。**不要加 bracket 栈、不要改链接/代码/autolink 逻辑。**

## 六、ArkTS 严格模式硬约束（8.2c-1 在此炸过 DevEco）

- **禁匿名对象字面量**：`scanDelims`/`processEmphasis` 的返回值与栈项全部用**具名 class + `new` + 字段赋值**（`Delimiter`、`EmphasisProcessor`）。**绝不** `return { numdelims: .., canOpen: .. }` 或 `this.delimiters = { cc, ... }`。
- **禁 `Record` / 对象属性索引访问**：`openersBottom` 是**数组** `(Delimiter|null)[]`，`openersBottom[obIndex]` 是数组下标（合法）。不要用 `obj[key]`。
- **禁 spread `{...}` / `any` / `unknown`**。
- **正则**：本批不要新增任何正则；标点/空白判定用上面的 charCode helper。
- 字符串取字符 `text[i]`、数组下标 `out[k]` / `openersBottom[idx]` 在本项目合规（现有代码即如此）。
- import 放文件顶部（`AstNode, AstNodeType` 已 import）。
- 提交前自检（Hermes 也会查）：`grep -nE "return \{|=> \{ *[a-z_]+:|: *\{ *[a-z_]+:" Inlines.ts` 抓匿名对象字面量应 **0 命中**（注意排除函数体 `{`）。

## 七、必过样例（实现后必须转正）

单字符 flanking：
- Ex351 `a * foo bar*` → `<p>a * foo bar*</p>`（开 `*` 后跟空格不能开）
- Ex352 `a*"foo"*` → `<p>a*&quot;foo&quot;*</p>`（`*` 前字母后标点，非左 flanking）
- Ex353 `* a *` → `<p>* a *</p>`（两侧空格）
- Ex360 `foo_bar_` → `<p>foo_bar_</p>`（`_` 词内不开）
- Ex361 `5_6_78` → `<p>5_6_78</p>`
- Ex362 `пристаням_стремятся_` → 字面（西里尔词内 `_`）
- Ex366 `*foo bar *` → `<p>*foo bar *</p>`（闭 `*` 前空格不能合）
- Ex374 `_foo_bar` → `<p>_foo_bar</p>`

嵌套 / strong / rule-of-three：
