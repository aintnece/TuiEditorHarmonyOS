# 9.3 — Phase 9 启动期/首帧主题同步修复 · CC 实施 spec

> 真机发现两个首帧主题 bug（Phase 9.2 真机 5 点已过，切换后均正常 → 订阅链路对，缺的是「初始值同步」）。验证：真机。

## 问题与根因
- **问题1：启动界面（`pages/Index.ets`）不跟随系统深色** —— Index.ets 全硬编码浅色（`#ffffff`/`#24292e`/`#f6f8fa`…），**根本没接 ThemeService**。
- **问题2：首次进文档时编辑区+预览浅色、外壳深色，切换一次才恢复** —— `EditorPage.ets` L385 `opts.useDarkTheme = false` 硬编码 → EditorCore `new HtmlRenderer({darkMode:false})` → 预览首帧浅色；且首帧 `this.theme` 疑似未取到 ThemeService 当前 Dark（MdEditor 全靠 @Prop themeColors，零硬编码，浅色只能是 themeColors 首帧=Light）。切换时 `this.theme = t` 重新赋值 + 订阅回调重建 renderer 才对齐。

---

## 修复 1 — Index.ets 接入 ThemeService（`pages/Index.ets`）

### 1.1 import + @State + 订阅
```ts
import { ThemeService, ThemeColors } from '../services/ThemeService';
// struct 内加：
@State private theme: ThemeColors = ThemeService.getInstance().getTheme();
// aboutToAppear 内加（系统/手动切换时启动页也跟随）：
ThemeService.getInstance().onThemeChange((t: ThemeColors): void => { this.theme = t; });
```
> Index 是 loadContent 的首页，app onCreate 已 init+setSystemColorMode，故 @State theme 首帧即当前主题。

### 1.2 硬编码颜色 → token（逐处替换）
| 位置 | 原值 | 改为 |
|---|---|---|
| 页面根 `.backgroundColor` | `#ffffff` | `this.theme.editorBg` |
| 标题栏 `.backgroundColor` | `#f6f8fa` | `this.theme.toolbarBg` |
| 标题栏 `.border` color | `#dfe2e5` | `this.theme.borderColor` |
| 'TUI Editor' `.fontColor` | `#24292e` | `this.theme.toolbarFg` |
| 新建按钮 `.backgroundColor` | `#0366d6` | `this.theme.toolbarActive` |
| 新建按钮文字 | (默认) | 显式 `.fontColor('#ffffff')`（accent 上的白字，深浅都白）|
| 文件项 `.fontColor` | `#24292e` | `this.theme.editorFg` |
| 文件项 `.border` color | `#f0f0f0` | `this.theme.borderColor` |
| 底部信息 `.fontColor` | `#959da5` | `this.theme.editorPlaceholder` |

> 全部用 `this.theme.*`，深浅由 ThemeService 决定。不新增硬编码。

## 修复 2 — EditorPage 首帧主题同步（`pages/EditorPage.ets`）

### 2.1 useDarkTheme 取当前主题（L385）
```ts
opts.useDarkTheme = false;
```
→
```ts
opts.useDarkTheme = ThemeService.getInstance().getTheme().isDark;
```
> 这让 EditorCore 初始 renderer 的 darkMode 正确 → 预览首帧不再浅色。必须在 `Editor.factory(opts)` 之前（本就在前）。

### 2.2 aboutToAppear 强制同步 this.theme（防御 @State 初始化时机）
在 `editorContext.editor = Editor.factory(opts);` **之后**（或 aboutToAppear 早段）加一行，确保首帧 @State theme = 当前主题：
```ts
this.theme = ThemeService.getInstance().getTheme();
```
> 即使 L45 `@State theme = getTheme()` 已设，此处重设保证首帧 build 前 theme 为最新 Dark；外壳/MdEditor/MdPreview 的 `themeColors: this.theme` 首帧即正确。已有 `onThemeChange`(L413) 处理后续变化，不动。

## ArkTS / 注意
- Index.ets 加 import ThemeService/ThemeColors；onThemeChange 回调用具名 `(t: ThemeColors): void`。
- 仅改颜色取值与 useDarkTheme/this.theme 同步，**不改布局结构、不动其他逻辑**。
- 不碰 ThemeColors 调色板值、不碰 9.2 已建的订阅链路。

## 真机验证
1. 系统深色下**冷启动 app** → 启动界面（文件列表页）即为深色（标题栏/背景/文字跟随）。
2. **首次点进文档**（不做任何切换）→ 编辑区 + 预览 + 外壳三处**首帧即一致**（系统深色则全深）。
3. 系统在 app 运行中切换亮暗 → 启动页 + 编辑页都跟随（userMode=系统时）。
4. 9.2 既有 5 点不回归。
